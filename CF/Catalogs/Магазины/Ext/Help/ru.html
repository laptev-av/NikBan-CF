Магазины

Информация о магазинах сети вносится в справочник Магазины. Магазином считается, как правило, территориально удаленная организационная единица предприятия, предназначенная для продажи товаров розничному покупателю. Товар также может отпускаться со складов магазина мелким оптом.

Список элементов справочника магазинов открывается по ссылке НСИ – Предприятие – Магазины. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные счета, задавая поиск по значениям различных реквизитов.

В справочнике описываются настройки для работы складов магазина: склады поступления и продажи, настройки ордерной схемы работы складов, настройки применения цен и скидок при розничных и мелкооптовых продажах. Можно также настроить автоматическое распределение товаров по складам при поступлении в магазин и распределение товаров по складам продаж (отделам торгового зала) для автоматического списания остатков при продаже через РМК.

Пример. В магазине «Обувь» имеется несколько торговых залов (складов): «Женская обувь», «Мужская обувь», «Спортивная обувь». В каждом торговом зале продается соответствующая номенклатурная группа товаров. Для каждой номенклатуры указывается ее принадлежность к номенклатурной группе. При поступлении товара можно автоматически оприходовать его в соответствующие торговые залы на основании настроенного распределения поступлений.

Новый элемент справочника можно ввести с помощью кнопки Создать. При этом откроется форма помощника создания магазина для заполнения основных реквизитов магазина. Далее нужно ввести наименование магазина. Реквизит площадь торгового зала позволяет оценить эффективность использования торговой площади.

Заполнение автоматически подставляемых значений склада поступления и склада продажи

В магазине может быть несколько различных складов, один из которых назначается складом поступления, а другой – складом продажи. Склад поступления подставляется по умолчанию в документы поступления и перемещения товаров из магазина в магазин, а склад продажи – в документы продаж в розницу или мелким оптом.

Например, в магазине «Одежда» все товары проходят предпродажную подготовку, поэтому приходуются на специальный склад «Предпродажная подготовка». Этот же склад используется для заполнения поля Склад документов, полученных из конфигурации «Управление торговлей».

Для склада продаж обязательно указывается организация продаж, подставляемая в документы по умолчанию. С этого склада автоматически списываются остатки товаров при продаже розничному покупателю в режиме РМК, если не настроено распределение товаров по складам для продаж.

Склад поступления может совпадать со складом продажи. В этом случае нужно оставить переключатель в положении Один склад.

Настройка использования ордерной схемы для складов магазина

Ордерная схема в магазинах используется, если фактические операции с товаром и оформление документов производятся в различные моменты времени разными пользователями.

Использование ордерной схемы можно настроить для приходных и расходных операций, установив соответствующий флажок Использовать ордерную схему при поступлении или Использовать ордерную схему при отгрузке. Использование ордерной схемы можно также настроить для оприходования излишков и списания недостач, установив флажок Использовать ордерную схему при отражении излишков недостач. Настройка будет действовать для всех складов магазина.

Перемещение товаров из магазина в магазин всегда производится по ордерной схеме, поэтому флажок параметра установлен по умолчанию и недоступен для изменения.

При безордерной схеме работы Менеджер магазина-отправителя формирует документ Перемещение товаров, указывая Магазин-получатель и перечисляя собранные к отправке товары. Документ передается в Магазин-получатель средствами РИБ. Если количество поступившего товара не совпадает с количеством, указанным в документе Перемещение товаров, то Менеджер магазина-получателя оформляет Акт о расхождении при приемке товаров. В противном случае никаких действий в программе не требуется.

Настройка параметров ценообразования в магазине

Для каждого магазина нужно указать правило ценообразования, согласно которому на продаваемую номенклатуру будет назначена розничная цена.

С помощью флажка Использовать применение цен можно управлять возможностью отложить применение на кассах ККМ назначенных централизованно розничных цен. При этом должна быть установлена функциональная опция Применение цен (Администрирование – Маркетинг). Затем нужно перейти к прочим настройкам магазина.

При продаже со скидкой рассчитанная цена может оказаться слишком низкой. Чтобы этого избежать, можно назначить вид минимальных цен продажи. При этом рассчитанная цена со скидкой автоматически заместится значением минимальных разрешенных цен. Подробнее см. раздел «Настройка видов цен номенклатуры».

Чтобы избежать повышенного расхода разменной монеты, можно установить одно из значений списка Порядок округления суммы чека в пользу покупателя. Возможно округлять итоговую сумму чека, либо построчно. Например, при округлении до рубля итоговой сумма чека, сумма по товарным строкам чека с учетом скидок, будет округлена вниз до ближайшего целого значения рублей. При этом «лишние копейки» будут вычтены из суммы позиции чека с максимальной суммой.

В разделе Адреса и телефоны нужно ввести телефоны и фактический адрес магазина. Также можно добавить несколько телефонов и факсов.

После создания элемента справочника Магазин, в основных настройках можно указать список номенклатуры, на которую не действуют никакие скидки (Сегмент, исключаемой из скидок номенклатуры). Например, это социально значимые товары, продающиеся со сниженной наценкой. Вне зависимости от условий предоставления скидки эти товары будут продаваться по назначенной розничной цене.

Помимо основных реквизитов элемента справочника Магазин можно создать набор дополнительных реквизитов и сведений.

Для этого нужно нажать кнопку Дополнительные сведения в командной панели списка магазинов и воспользоваться командой Изменить состав дополнительных сведений для магазина. В открывшейся форме можно подобрать нужные реквизиты из числа уже созданных или дополнить список нужным реквизитом.

Склады магазина

Для создания нового склада следует перейти в список Склады панели навигации карточки магазина и нажать кнопку Создать. В открывшейся форме ввести наименование и выбрать тип склада: Складское помещение или Торговый зал.

Склады с типом Торговый зал могут создаваться в целях обособления учета товародвижения. Например, физически торговый зал может быть представлен единым помещением, но в нем выделено несколько зон учета (отделы), представленных в конфигурации в качестве складов.

Нужно указать организацию, от имени которой по умолчанию будут оформляться документы. Для склада с типом Торговый зал этот реквизит обязателен. В соответствии с этим реквизитом и распределением продаж по складам магазина производится автоматическое формирование чеков продажи при использовании РМК. Подробнее см. «Оформление розничных продаж и возвратов в интерфейсе РМК».

Если продажи со склада облагаются единым налогом на вмененный доход, то нужно создать запись в списке Применения ЕНВД для данного склада. При этом дата автоматически заполнится первым днем текущего месяца и в поле организации – значением организации склада.

Если по каким-то причинам действие ЕНВД на данном складе отменено, нужно создать новую запись с помощью копирования записи введения ЕНВД на данном складе и, установив дату планируемого окончания действия ЕНВД, снять флажок Розничная торговля облагается ЕНВД.

Распределение поступлений

В программе предоставляется возможность автоматического распределение поступления и продажи товаров по отделам магазина. Сервис распределение товаров вызывается командой из документов Поступление товаров и Перемещение товаров, Реализация товаров. Кроме того, правила распределения продаж товаров и услуг применяются при формировании документов Чек ККМ при оформлении розничных продаж в режиме РМК. Настройки сервиса можно выполнить, перейдя по ссылкам Распределение поступлений, Распределение продажи и Распределение продаж услуг из панели навигации карточки магазина.

Если в магазине указан склад поступления и склад продажи, то все товары по умолчанию будут приходоваться/списываться с указанного склада. С помощью схемы распределения товаров по складам можно задать уточнение распределения по всем или некоторым номенклатурным группам.

Для настройки схемы распределения поступления товаров по складам магазина нужно воспользоваться пунктом Распределение поступления панели навигации карточки магазина.

В строках таблицы распределения показываются номенклатурные группы товаров, а в столбцах – все склады магазинов, кроме указанного в качестве основного. Для нужной номенклатурной группы необходимо установить флажок в ячейке соответствующего склада.

Для уточнения распределения продаж необходимо перейти к форме настройки по ссылке Распределение продаж в панели навигации карточки магазина.

Таблицу распределения продаж удобно заполнять, воспользовавшись командой Заполнить по распределению поступлений. При такой схеме списание остатков товара будет производиться со складов, на которые он был начислен при поступлении. Можно также назначить склад списания товаров для определенного рабочего места кассовой линейки. В этом случае остатки товара при оформлении продажи на данной кассе будут списаны с указанного склада, в то время как на любом другом рабочем месте продажи будут производиться с общего склада продажи или со склада, указанного для номенклатурной группы.

Пример. Сигареты и зажигалки размещены в прикассовой зоне всех касс магазина, три из которых находятся на первом этаже в отделе продажи продуктов питания, а «Касса 4» – в отделе хозтоваров полуподвальном помещении. В этом случае для номенклатурной группы «Сигареты», включающей в себя сигареты и зажигалки, устанавливается склад продажи «Продукты», а для рабочего места «Касса 4» – склад «Хоз. товары».

С помощью распределения продаж услуг по организациям можно уточнить распределение продаж некоторых услуг при продаже товаров определенных номенклатурных групп.

Так, при продаже услуги Доставка мебели, отнесенная к номенклатурной группе Мебель, может быть отнесена к ИЧП «Предприниматель», в то время как товары номенклатурной группы Мебель будут списываться со склада «Мебель» и организации склада ТД «Комплексный».

График работы магазинов

Чтобы указать график работы магазина на следующий год, можно создать в списке графики работы магазина.

В открывшейся форме можно выбрать вид графика – пятидневка, шестидневка или полный и скорректировать тип дня. Для изменения типа дня двойным щелчком на нужной дате необходимо вызвать контекстное меню и выбрать одно из значений:Праздник, Предпраздничный, Рабочий, Суббота, Воскресенье. График работы магазина можно распечатать.

Графики работы магазина используются, чтобы спланировать потребности в товарах при снабжении магазина из распределительного центра. Подробнее см. раздел «Формирование заказов с помощью сервиса «Анализ продаж и формирование заказов»

Интервалы работы магазина

В интервале работы магазина описывается режим рабочего дня магазина:начало и окончание работы сотрудников магазина, количество сотрудников и состав работ, которые необходимо выполнить в этот день.

В системе имеется возможность настроить несколько интервалов работы магазина для будничных, праздничных или предпраздничных дней, а также любой нестандартный режим работы.

Выплаченная зарплата

В списке Выплаченная зарплата выводятся документы, по которым сотрудникам была выплачена заработная плата, с указанием ее размера для каждого сотрудника, номера и даты проведения документа на расход денежных средств и номера, и даты проведения ведомости на выплату заработной платы.

По двойному щелчку левой кнопки мыши по строке таблицы можно просмотреть соответствующий данной транзакции расходный кассовый ордер.

Поставляемая номенклатура

В системе существует возможность просматривать список единиц номенклатуры, поставляемых той или иной организацией. Для каждой записи указывается магазин, в который совершаются поставки, наименование организации, осуществляющей поставки, наименование номенклатуры, ее характеристика, цена и дата последнего поступления.

Таблицу со списком номенклатур можно обновлять автоматически за указанный период на основе документов Заказ поставщику, Поступление товаров, Заказ на перемещение и Перемещение товаров. Окно выбора параметров обновления открывается по нажатии кнопки Обновление номенклатуры поставщиков.
