Правила отбора товаров для проведения инвентаризаций

Справочник Правила отбора товаров предназначен для фиксирования правил, по которым могут быть сформированы списки товаров для проведения выборочных инвентаризаций.

Настроенные правила отбора пользователь может использовать в документе Приказ на проведение инвентаризации.

Список настроенных правил отбора открывается по ссылке Склад – См. также - Правила отбора товаров. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные правила, задавая поиск по значениям различных реквизитов.

Новое правило отбора можно добавить с помощью кнопки Создать в командной панели списка. Создавая новый элемент справочника Правила отбора товаров, администратор базы данных вводит название правила, выбирает и настраивает схему компоновки данных из числа предоставляемых системой или создает свою. В программе доступны следующие схемы компоновки данных:

· Обнуление остатка – в список войдут товары, у которых учетный остаток обнулился в период, начиная с указанной пользователем даты до момента формирования списка товаров. Схема имеет параметр Начало периода;

· Номенклатурная группа – пользователь может выбрать номенклатурную группу товаров, по которой нужно провести инвентаризацию;

· Дорогие товары – в список войдут товары, розничная цена которых на момент формирования списка вышеуказанной пользователем в параметре Цена товара. Пользователь может также исключить из списка товары, проверенные при других инвентаризациях за указанный период с Даты начала до момента формирования списка;

· Списанные товары – в список товаров войдут товары, списанные по документам списания с указанной пользователем аналитикой хозяйственной операции за указанный пользователем период с даты начала периода до момента формирования списка. Так же как и в схеме по дорогим товарам, пользователь может исключить из списка товары, проверенные при других инвентаризациях за тот же период;

· Отрицательные остатки – в список для инвентаризации войдут товары с отрицательным учетным остатком на момент формирования списка;

· Попытки продаж, превышающих остаток – правило, аналогичное отрицательным остаткам в случае, если для магазина запрещены продажи количества, превышающего остаток (т. е. установлены флажки Контролировать остатки товаров на складе в справочнике Дополнительные настройки пользователя для всех пользователей, оформляющих продажи, и Контролировать остатки товаров при закрытии чека в справочнике Настройки РМК при использовании режима РМК). При установленной настройке Фиксировать попытки продаж, превышающих остаток на странице Продажи раздела Администрирование информация о таких случаях сохраняется в программе, и на ее основе формируется список товаров для инвентаризации;

· Товарная категория – пользователь может выбрать товарную категорию, по которой нужно провести инвентаризацию.

Назначаемый по умолчанию параметр отбора Не выводить пересчитанные товары данного приказа предназначен для того, чтобы в каждый последующий пересчет не добавлялись товары из предыдущих, то есть те, которые уже были пересчитаны ранее.
