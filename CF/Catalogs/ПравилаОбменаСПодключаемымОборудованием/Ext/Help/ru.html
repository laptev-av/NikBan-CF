Справочник "Правила обмена с подключаемым оборудованием offline"

Правила обмена используются при работе с кассами ККМ и электронными весами, которые работают в режиме OffLine (доступны периодически). С помощью правил обмена настраивается список товаров для выгрузки данных в кассу ККМ и электронные весы, назначаются соответствующие коды для работы с подключаемыми устройствами в удаленном режиме.

· Для каждой кассы должно существовать минимум одно правило. Одно и тоже правило может быть назначено нескольким кассам.

· Для каждых весов должно существовать минимум одно правило. Одно и то же правило может быть назначено нескольким весам.

Список элементов открывается по ссылке Администрирование – Подключаемое оборудование - Правила обмена с подключаемым оборудованием. С помощью команды Ctrl+F можно отфильтровать список по любой комбинации символов. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные строки, задавая поиск по значениям различных реквизитов.

При создании нового правила обмена выбирается тип подключаемого оборудования и склад, на котором установлено данное оборудование.
При подготовке данных для выгрузки на оборудование можно использовать различные фильтры. Может быть установлен отбор номенклатуры для выгрузки:по группам номенклатуры, упаковкам, определенным сегментам номенклатуры. Можно также выгрузить только товары с остатками по складу, товары, разрешенные к продаже или с определенной ролью в ассортименте.

Для товаров, выгружаемых в кассу программа автоматически присваивает код, а в правиле можно ограничить список товаров, указав "Максимальный код". В правиле можно установить отбор номенклатуры, например, можно загружать только те позиции номенклатуры, которые имеются в наличии на указанном складе (флажок "Только имеющиеся на складе").
 

После заполнения правил отбора номенклатуры необходимо сформировать список товаров для выгрузки с помощью кнопки Записать и закрыть.

 

При локальном формировании SKU в пределах одного узла РИБ (общая настройка Коды товаров SKU назначаются в главном узле РИБ отключена) для правил обмена с ККМ Offline в командной панели сформированного списка товаров становится доступна кнопка Создать SKU для товаров. Следует учесть, что созданные коды в этом случае не мигрируют по сети и не могут использоваться для идентификации товаров на кассах других магазинов.

Заполнение карточки правил для весов

В карточке правила обмена необходимо задать наименование правила, в поле "Тип подключаемого оборудования" выбрать значение "Весы с печатью этикеток", обязательно указать склад. Для товаров, выгружаемых в весы программа автоматически присваивает код (PLU), а в правиле можно ограничить список товаров, указав "Максимальный код". Так же необходимо указать "Префикс весового товара".

 

Префикс устанавливается в качестве второй цифры штрихкода (слева) весового товара. Первая цифра штрихкода весового товара всегда двойка. Таким образом, по первым двум цифрам штрихкода касса может различать весовой и штучный товар. О значении, которое должно быть установлено в поле "Префикс весового товара" более подробно см. документацию к кассе или программному обеспечению, установленному на кассу. Если необходимо сформировать локальные в пределах правил обмена с оборудованием коды PLU, то необходимо установить флажок Своя нумерация PLU на оборудовании и указать максимальный код PLU, допустимый для данной модели весов.

Выполнение обмена с оборудованием Offline

Обмен с подключаемым оборудованием производится с помощью команд закладок ККМ Offline и Весы с печатью этикеток в списке Продажи – Розничные продажи - Обмен с подключаемым оборудованием.

 

В списках отображаются все устройства, и для каждого из них – правило обмена, склад размещения. Для касс Offline дополнительно выводится касса ККМ, с которой связано устройство.

С помощью команды Выгрузить товары можно выгрузить обновление справочника номенклатуры:новые товары, товары с новыми характеристиками и упаковками, а также товары, у которых изменились розничные цены.

Команда Полная выгрузка может выполняться длительное время, т. к. выгружаются все номенклатурные позиции, которые удовлетворяют отбору правил обмена.

После успешной операции выгрузки товаров в таблице устанавливается дата и время последней произведенной выгрузки.

Предусмотрена также кнопка очистки памяти устройства .

С помощью команды Загрузить отчет производится загрузка данных из таблицы продаж, сформированной на ККМ Offline. В соответствии с данными о продажах формируется документ Отчет о розничных продажах, учитывающий продажи и возвраты товаров, произведенные в течение одной кассовой смены. В поле Комментарий документа будет автоматически сформирована запись о произведенной загрузке: указано рабочее место и драйвер оборудования.

Помимо команд управления выгрузкой и загрузкой предоставляется возможность просмотреть список товаров, сформированный для выбранного устройства, и открыть для редактирования установленное правило обмена.
