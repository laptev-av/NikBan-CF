Виды номенклатуры

Справочник Виды номенклатуры – классификатор, который используется для обобщения свойств товаров и услуг. Например, с помощью вида номенклатуры можно задать уровень детализации ведения товарных остатков – в разрезе номенклатуры или характеристик номенклатуры.

Список элементов открывается по ссылке НСИ – Номенклатура – Виды номенклатуры. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные документы, задавая поиск по значениям различных реквизитов.

Пользователь может добавить в справочник неограниченное количество видов номенклатуры, однако все они могут быть только трех типов: товар, услуга, подарочный сертификат.

Новый элемент справочника можно ввести с помощью кнопки Создать.

Для номенклатуры может быть определено использование характеристик:

· Не использовать – учет по характеристикам не ведется;

· Использовать общие для вида номенклатуры – характеристики подчинены данному виду номенклатуры;

· Использовать индивидуальные для номенклатуры – характеристики подчинены номенклатурным позициям.

После выбора типа номенклатуры и использования характеристик следует записать элемент справочника (кнопка Записать объект или Ctrl+S). При этом автоматически сформируется запись о наборе свойств данного вида номенклатуры. Если установлено использование характеристик, набор свойств создастся и для характеристик вида номенклатуры. Щелчком мыши на значке просмотра можно открыть форму настройки наборов дополнительных реквизитов и сведений.

Для учета продаж алкогольной продукции необходимо заполнить Сведения об алкогольной и спиртосодержащей продукции.

Для каждого вида номенклатуры могут быть определены Шаблоны ценника и этикетки.

Для подключения возможности учета по сериям нужно установить флажок Использовать серии для выбранного вида номенклатуры. При этом в форме появится поле Серии идентифицируют, в котором необходимо выбрать, что будет идентифицировать серия для данного вида товаров:

· экземпляры товаров, имеющих уникальные серийные номера, например, для бытовой техники и электроники;

· партии товаров, имеющие одинаковые номера серий, – для товаров серийного производства, например, лекарственных препаратов;

· партии товаров, имеющие одинаковые сроки годности, – для товаров, имеющих определенную дату окончания реализации;

· партии товаров, имеющие одинаковые номера серий и сроки годности, – для учета и номера серии, и даты окончания реализации.

Пример. На предприятии ведется контроль серийных номеров бытовой техники при реализации, чтобы можно было контролировать их при возврате или приемке в сервис для ремонта. В этом случае для вида номенклатуры «Бытовая техника» нужно указать, что серия идентифицирует экземпляры товаров, имеющих уникальные серийные номера.

Определение агентских услуг

Определение суммы выручки при продаже агентских услуг производится с помощью особого Вида номенклатуры.

Для вида номенклатуры с типом номенклатуры Услуга добавлена возможность указания признака Агентские услуги.

Если для определенной номенклатуры выбран вид номенклатуры с признаком Агентские услуги, то становится доступно обязательное к заполнению поле Договор платежного агента. При продаже номенклатуры с признаком Агентские услуги, сумма выручки учитывается в программе отдельно. Таким образом, вся выручка магазина делится учитывается по видам выручки - собственная и агентская в разрезе договоров.

Регистрация видов номенклатуры для учета товаров, подлежащих маркировке

Для маркируемых товаров предусмотрен отдельный тип номенклатуры – Продукция, маркируемая для ГИСМ. При вводе нового вида номенклатуры необходимо выбрать этот тип номенклатуры.

При указании номенклатуры такого типа автоматически устанавливается вариант идентификации серий – Серии идентифицируют Экземпляр товара.

Для регистрации КиЗ необходимо ввести вид номенклатуры с типом товара - Контрольный (идентификационный) знак. В состав реквизитов, которые используются для идентификации серии, должна обязательно входить информация о контрольном идентификационном знаке (его вид и номер). Дополнительно задается номер серии и указывается возможность сохранения в ИБ данных RFID-метки. Все остальные параметры задаются обычным образом.
