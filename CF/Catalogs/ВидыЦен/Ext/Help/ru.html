Виды цен

Для разграничения цен разных каналов сбыта или розничных цен магазинов используются различные виды цен. Виды цен также используются, если необходимо разграничить доступ пользователей к управлению ценами по товарным категориям.

Пример.
1. В магазине производится продажа товара мелким оптом. Цены продажи мелким оптом отличаются от цен продажи в розницу в том же магазине. В этом случае можно использовать виды цен «Мелкооптовая» и «Розничная».
2. Розничные цены магазинов одного региона отличаются от розничных цен магазинов другого региона. Виды цен: «Розничная цена Москва» и «Розничная цена Урал».
3. Цены на одежду и обувь формируют разные менеджеры центрального офиса. Виды цен:«Розничная цена Обувь» и «Розничная цена Одежда».
4. На часть номенклатуры цены назначает менеджер магазина, остальные цены управляются менеджером центрального офиса. Виды цен:«Цена центрального офиса», «Локальная цена магазина № 1». Причем для локальных цен магазинов заводится отдельный вид цен для каждого магазина.

Список видов цен можно открыть по ссылке Маркетинг - Ценообразование - Виды цен. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные строки, задавая поиск по значениям различных реквизитов.

Новую запись в справочник добавить с помощью кнопки Создать в командной панели списка.

Заполнение основных реквизитов

При создании нового вида цен пользователь указывает Наименование цены, которое должно достаточно полно отражать ее назначение.

Идентификатор для формул необходим при описании формулы расчета цены. Идентификатор формируется автоматически при вводе наименования цены и может быть откорректирован пользователем.

Флажок Цена включает НДС управляет включением суммы НДС в цену номенклатуры.

C помощью флажка Использовать при продаже управляется список цен для выбора вида цены при реализации товара мелким оптом или в розницу.

Виды цен, отмеченные флажком Использовать при передаче между организациями, используются при формировании документов Передача товаров между организациями предприятия.

Правила калькуляции цены

Можно выбрать следующие способы задания цены:

· задавать вручную;

· заполнять по данным информационной базы;

· заполнять по данным ИБ при поступлении;

· рассчитывать по другим видам цен.

Задавать вручную

Если выбран такой способ задания цен, то в документе Установка цен номенклатуры пользователь сможет вручную ввести цены в соответствующей колонке табличной части.

Заполнять по данным ИБ

При использовании этого способа цены в документе будут заполнены значениями, хранящимися в информационной базе. Для описания схемы заполнения значений вида цен данными информационной базы пользователь может выбрать одну из доступных схем:

· Цены поступления – анализируются цены поступления товаров в зависимости от установленных параметров: коэффициента упаковки и вида выбора цен – минимальные, максимальные, средние. При редактировании этой схемы СКД (в режиме толстого клиента) пользователь может определить значения параметров по умолчанию, а также установить ограничения на изменение значений по умолчанию пользователю, производящему перерасчет цен.

· Себестоимость – последняя запись регистра Себестоимость. При совместной работе с конфигурацией «Управление торговлей», редакция 11.0, заполняется при обмене рассчитанной себестоимостью с НДС. При автономном использовании конфигурации «Розница» в качестве себестоимости используется цена с НДС последнего поступления.

· Минимальные, максимальные или средние цены поставщиков – схемы расчета цен, основанные на определении максимальных, минимальных или средних цен на данную номенклатуру, зарегистрированных при последнем поступлении от разных поставщиков и в разные магазины.

· Произвольный.

С помощью кнопки Редактировать пользователь в режиме толстого клиента может вызвать форму Настройка схемы компоновки данных для вида цены.

Заполнять по данным ИБ при поступлении

При использовании этого способа цены в документе будут заполнены значениями, хранящимися в информационной базе, как и при использовании простого заполнения по данным информационной базы, однако в систему заносятся цены поставщика при поступлении. Такой способ заполнения цен используется, если условия договора не предусматривают жесткого контроля цен. В таком случае, как правило, в качестве стратегии розничного ценообразования применяется стратегия следования ценам поставщика. То есть центральный офис определяет правила расчета цены в виде розничной наценки, которая применяется к цене поступления товара в магазин. Цены рассчитываются при поступлении товара непосредственно в магазин – это один из этапов предпродажной подготовки товаров перед выкладкой в торговый зал.
Настройка схемы будет такой же, как и для способа Заполнить по данным ИБ.

Рассчитывать по другим видам цен

Способ используется, если вид цен имеет определенную зависимость от другого вида цен, выраженную формулой. Формулу можно ввести в поле вручную или воспользоваться конструктором формул.
Конструктор позволяет наглядно описать правило расчета цены с помощью функций, арифметических и логических операторов. В качестве переменных формулы расчета используются доступные для пользователя виды цен. Синтаксис формулы расчета можно проверить, нажав кнопку Проверить формулу на командной панели. Если формула составлена неверно, пользователю будет выведено сообщение об ошибке.
По умолчанию формула задается для всех ценовых групп номенклатуры. Но пользователь может ограничить ее действие, добавляя в табличную часть ценовые группы номенклатуры из справочника Ценовые группы и назначая для каждой из них свою формулу расчета цены.

Правила округления цены

Для каждого вида цен могут быть заданы правила округления цены, единые для всех цен или по диапазонам в зависимости от величины цены.

Для описания единого правила округления для всех цен нужно на странице Правила округления цены с помощью кнопки Добавить создать одну строку. При этом в поле Ценовой диапазон появится надпись <все цены>.

Чтобы поддержать схему округления по диапазонам цен нужно создать соответствующие строки, указав нижнюю границу каждого диапазона цен. Затем выбрать нужную точность округления для каждой строки.

Для того, чтобы получившиеся значения цен выглядели единообразно, можно также определить последнюю значащую цифру цены в колонке Психологическое округление, или воспользоваться Конструктором психологического округления.

С помощью переключателя внизу страницы пользователь может установить метод округления:По арифметическим правилам или Всегда в большую сторону.

Пример.
В магазине "Три толстяка" приняты следующие правила округления цен. В диапазоне:
- от 0 до 3 руб округление цены не производится;
- от 3 до 10 руб. – до 30 коп.;
- от 10 до 300 руб – до 1 руб.;
- свыше 300 – до 10 руб.
Создаем 4 диапазона цен, указывая в качестве нижней границы 3, 10 и 300. В первой строке округления нет, в следующих строках указываем точность округления 1, 10 и 100 и психологическое округление 0.3, 1 и 10 соответственно.
Проверяем результат с помощью Конструктора психологического округления.

Пороги срабатывания

Чтобы уменьшить количество переоценок при незначительных изменениях цен поставщиков, можно назначить Порог срабатывания:изменение в процентах старой цены, превышение которого вызовет перерасчет цен. Пороги срабатывания могут быть разными для различных ценовых групп.

Пример.
Цены закупки рассчитываются по цене, указанной в документе поступления. Причем по всем категориям товаров порог срабатывания равен 5%, по ценовой группе «Молоко» – 2 %, по ценовой группе «Хлеб» – 3 %.

Конструктор психологического округления.

Конструктор предоставляет возможность наглядным образом сформировать правила психологического округления, устанавливая значение значащей цифры для указанного разряда. Пользователь может проверить как работает правило, вводя произвольную цену, система отобразит цену, округленную по установленному правилу.
