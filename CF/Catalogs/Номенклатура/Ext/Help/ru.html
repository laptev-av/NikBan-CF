Номенклатура

Справочник номенклатура является центральным справочником торговой системы. В нем хранится вся информация, необходимая для заказа, оприходования и продажи товара, находящегося в ассортименте сети. Для структурирования данных и уменьшения количества ошибок при ручном вводе информации используется ряд вспомогательных справочников, настроенных заранее:

· Классификатор единиц измерения,

· Справочник упаковок,

· Шаблоны ценников и этикеток,

· Справочник видов номенклатуры.

Список элементов справочника открывается НСИ (Нормативно-справочная информация) – Номенклатура.

С помощью сервиса Повторная активация подарочного сертификата можно использовать повторно сертификаты:по истечении срока давности, т.е. списанные или проданные и погашенные, сертификаты определенной серии.

С помощью кнопки Создать группу Пользователь может добавить иерархию в дерево номенклатуры.

С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные документы, задавая поиск по значениям различных реквизитов.

Новый элемент справочника можно ввести с помощью кнопки Создать, в списке номенклатуры (или Все действия - Создать). Можно также создать новую карточку на основании уже имеющейся (Все действия - Скопировать), при этом реквизиты нового элемента заполнятся в соответствии с выбранной позицией. В карточку новой позиции номенклатуры вводится следующая информация:

· Наименование – название товара или услуги, которое используется для быстрой визуальной идентификации номенклатуры в процессе работы с программой;

· Полное наименование – наименование товара и услуги, которое будет печататься во всех документах;

· Артикул – артикул позиции номенклатуры;

· Ставка НДС – устанавливается в соответствии с законодательством;

· Единица измерения – единица измерения, в которой ведется учет складских остатков товаров и указывается количество номенклатуры в одной упаковке. Для заполнения единицы измерения используется справочник Классификатор единиц измерения;

· Группа – группа, в которую входит данная позиция номенклатуры в соответствии с классификацией (иерархией) товаров в справочнике Номенклатура;

· Вид номенклатуры – указывается вид, к которому следует отнести данную позицию номенклатуры. При выборе вида номенклатуры автоматически заполняются параметры данного вида номенклатуры:тип номенклатуры (товар или услуга), использование характеристик, использование упаковок. Изменение этих параметров невозможно;

· Ценовая группа – ценовая группа, в которую входит позиция номенклатуры. В разрезе ценовых групп номенклатуры устанавливаются правила ценообразования;

· Номенклатурная группа – используется для распределения товара по складам магазина. Поле заполняется значениями справочника Номенклатурные группы;

· Набор упаковок – указывает на способ упаковки товаров при поступлении. Поле заполняется значениями справочника Упаковки;

· Страна происхождения – выводится на ценник товара, размещаемый в торговом зале. Заполняется значениями классификатора Страны мира;

· Вес нетто – для номенклатурной позиции может быть указан вес без учета веса упаковки;

· Весовой товар – отмечается флажком;

· Описание товара – в карточке также может быть размещено текстовое описание товара.

После ввода основных сведений о позиции номенклатуры необходимо нажать кнопку Записать.

Чтобы добавить в карточку номенклатуры изображение, нужно добавить файл изображения в хранилище файлов (Органайзер – Файлы). При добавлении необходимо для реквизита Папка выбрать значение типа данных Позиция номенклатуры и нужную позицию из справочника Номенклатура. Для добавления в карточку номенклатуры картинки необходимо кликнуть по надписи Добавить изображение и выбрать файл картинки.

После записи основных данных для номенклатуры могут быть введены дополнительные данные: характеристики, упаковки, штрихкоды и т.д.

Дополнительные реквизиты номенклатуры

Список дополнительных реквизитов располагается на закладке Дополнительные реквизиты.

Для заполнения значения дополнительного реквизита нужно нажать кнопку выбора в поле ввода реквизита. В зависимости от типа значения, указанного для свойства, открывается соответствующая форма: справочник значений свойств объекта, поле для указания даты, поле ввода числа, справочник клиентов и т.д. Если кнопки выбора нет, это означает, что поле предназначено для ввода текстовой информации. Следует ввести значение нужного типа или выбрать нужное значение из соответствующего справочника.

Список дополнительных сведений открывается при нажатии соответствующей пиктограммы или выборе действия Дополнительные сведения из списка номенклатуры или из формы объекта. В списке указаны сведения, определенные в наборе свойств для вида номенклатуры, либо определенные для данной номенклатурной позиции из числа общих свойств.

Характеристики номенклатуры

Если для данной позиции установлено ведение характеристик, общих для вида номенклатуры, то просмотреть список характеристик можно с помощью пункта Характеристики панели навигации карточки номенклатуры. Отображаемый список формируется в справочнике Виды номенклатуры, но пользователь имеет возможность ввести недостающую характеристику непосредственно в карточке номенклатуры. Для этого нужно воспользоваться кнопкой Создать и ввести значения дополнительных реквизитов характеристики в открывшейся форме.

Аналогично создается индивидуальная характеристика номенклатурной позиции, если она принадлежит виду номенклатуры с ведением характеристик, индивидуальных для номенклатуры.

Создание иерархии справочника номенклатуры

Чтобы создать новую группу номенклатуры выберите команду Создать на командной панели списка номенклатуры, введите наименование и выберите группу номенклатуры, в которую войдет создаваемая группа.

Заполнение информации об упаковках товара

Список упаковок, входящих в выбранный набор, можно просмотреть из пункта меню Упаковки панели навигации карточки номенклатуры. В случае индивидуального набора упаковки необходимые упаковки можно создать непосредственно в форме списка.

Если товар может поставляться в магазин в различной упаковке, то для позиции номенклатуры определяется используемый Набор упаковок. Если для позиции номенклатуры используются индивидуальные упаковки, то из справочника Наборы упаковок выбирается предопределенный элемент Индивидуальный набор упаковок.

Введение информации о составе комплекта

Является ли товар комплектом, определяется только тем, заполнен ли для него список комплектующих (пункт Комплектующие панели навигации документа), из которых он состоит.

Для ввода комплектующих с помощью команды Создать выберите номенклатуру, входящую в состав. Для каждой комплектующей указывается:

· Характеристика, если для нее установлено ведение характеристик;

· Упаковка;

· Количество упаковок;

· Доля стоимости - весовой коэффициент учета стоимости комплектующей в составе комплекта.

Введение информации о штриховом коде

Каждой номенклатуре с учетом характеристики можно сопоставить Штриховой код, по которому товар будет идентифицироваться при продаже на рабочем месте кассира. (РМК).

Для того, чтобы ввести новый штрихкод перейдите к списку Штрихкоды в списке команд панели навигации карточки номенклатуры и выберите команду Создать
В открывшейся форме выберите значение Характеристики номенклатуры, упаковки и Тип штрихкода. Переведите курсор в поле штрихкода и считайте его сканером или введите цифры вручную.

Штриховой код типа EAN128 имеет символы, разделяющие код на группы цифр. При считывании кода сканером разделительные символы опускаются. Чтобы вывести разделительные символы на печать их нужно внести в поле Представление кода.

Отдельный штрихкод можно создать и для каждой упаковки номенклатуры. Тип штрихкода упаковки может отличаться от типа штрихкода номенклатуры с определенной характеристикой. Например, для товарной позиции используется EAN13, а для упаковки EAN128.

Создание PLU кода номенклатурной позиции

PLU-код представляет собой цифровой код продажи номенклатуры и ведется в разрезе упаковок и характеристик номенклатурной позиции. Для каждого сочетания упаковки и характеристики позиции PLU- код создается автоматически при сохранении карточки номенклатурной позиции.

Создание элемента Подарочный сертификат

Для регистрации изготовленных сертификатов нужно создать Вид номенклатуры с типом Подарочный сертификат и ввести новый элемент справочника Номенклатуры с этим видом.

Для каждого сертификата создается PLU-код, который передается на кассу ККМ для регистрации продажи. Его можно просмотреть и по необходимости отредактировать, воспользовавшись пунктом меню Коды товаров PLU.

Помимо кода PLU сертификат может иметь штрихкод для его идентификации с помощью сканера штрихкодов. Перейдя на закладку Штрихкоды, пользователь может создать список штрихкодов для данного сертификата определенного типа.

Выпущенные сертификаты могут иметь индивидуальные серийные номера. Чтобы ввести в систему серийные номера нужно на закладке Сертификаты установить флажок Использовать серийные номера и указать тип используемого кода: магнитный, штриховой, смешанный. При этом станет доступна табличная часть Серийный номер, в которую пользователь может добавить записи вручную или воспользоваться обработкой Заполнить серийные номера.

Пользователь вводит Наименование Подарочного сертификата, указывает единицу измерения и ставку НДС 18%. Перейдя на страницу Сертификаты, вводит Номинал и Срок действия сертификата. В системе поддерживаются бессрочные сертификаты, сертификаты, заканчивающие свое действие на определенную дату и действующие в течение определенного количества дней после продажи.

Повторная активация подарочного сертификата

По истечении срока давности использованные, т.е. списанные или проданные и погашенные, сертификаты определенной серии можно использовать повторно. Для этого их нужно активировать с помощью сервиса Повторная активация подарочного сертификата, расположенного в списке Сервис командной панели справочника номенклатуры (НСИ (Нормативно-справочная информация) – Номенклатура) или на закладке Сертификаты номенклатурной позиции, соответствующей сертификату. Активация подарочных сертификатов доступна пользователям с дополнительным профилем прав доступа Ответственный за ведение номенклатуры.

Пользователь может ввести в табличную часть сертификаты с помощью сканера штрих кодов или считывателя магнитных кодов, а также добавив новую строку вручную с помощью команды Добавить или клавишей Insert. В поле наименование пользователь может ввести символы или с помощью клавиши F4 открыть форму выбора номенклатуры с предустановленным отбором по подарочным сертификатам. Удобно, чтобы в форме выбора был установлен режим просмотра Список. После выбора сертификата необходимо выбрать номер подарочного сертификата из справочника серийных номеров. Выбор осуществляется с отбором по выбранному сертификату-владельцу. При ручном вводе система проверяет, что номер сертификата принадлежит выбранному сертификату, сертификат с данным номером погашен и вводится в данный документ в первый раз. При формировании таблицы с помощью оборудования также производится контроль дублирования строк.

Внесенные в табличную часть сертификаты активируются с помощью команды Повторно активировать. При этом в табличной части появляется отметка в колонке Активирован. Отметка Повторно активирован также появляется в списке на закладке Сертификаты карточки номенклатуры. С вновь активированными сертификатами можно производить любые операции складского учета:оприходование, перемещение со склада на склад и списание.

Алкогольная продукция

В процессе создания номенклатурной позиции при указании вида номенклатуры с флагом Алкогольная продукция, пользователю предоставляется возможность ввести информацию, необходимую для заполнения Журнала учета объемов розничных продаж алкогольной продукции:

· Вид алкогольной продукции в соответствии со справочником,

· Объем единицы розничной продажи номенклатуры в декалитрах (1 декалитр = 10 литрам),

· Производителя или импортера алкогольной продукции. Причем поля Производитель и Производитель (импортер) алкогольной продукции может иметь различные значения как отечественной, так и для импортной продукции.

Ввод информации о КИЗ в информационную базу

Информация о контрольно идентификационных знаках вводится в справочник Номенклатура.

Для регистрации контрольно идентификационных знаков (КиЗ) используется тип номенклатуры Товар (Контрольный (идентификационный) знак) и политика учета серий – Справочное хранение серий. Для серий, которые используются для описания КИЗ необходимо указание номера КИЗ и использование RFID-метки. В рамках созданного вида номенклатуры регистрируется информация о КиЗ.

В карточку КИЗ вводятся следующие дополнительные сведения:

· Вид. Разделение КИЗ по характеру крепления. КИЗ могут быть вшивные, клеевые и навесные, по конструкции похожие на пластиковую пломбу. Во всех случаях при отсоединении КИЗ от товара обязательно должна нарушаться его физическая целостность, для исключения его повторного использования.

· Способ выпуска в оборот. Определяет, в какой момент производится маркировка товаров:при производстве товаров на территории таможенного союза или при импорте товаров из стран, не входящих в ЕАЭС. Для маркировки товаров, которые произведены странами ЕАЭС применяются КиЗ зеленого цвета, а товары импортных производителей маркируются КиЗ красного цвета.

· Размер. Этикетка, на которую нанесен КиЗ, может иметь следующие размеры: 25 мм x 160 мм и 53 мм x 80 мм. Контрольно идентификационный знак может быть заказан для конкретной позиции номенклатуры (персонифицированный КиЗ). В этом случае в карточке КиЗ указывается номер GTIN для конкретного товара. Предусмотрена возможность с помощью одной позиции КиЗ, зарегистрированной в списке Номенклатура, заказать КиЗ сразу для некоторых товаров. В этом случае для КиЗ должен быть включен учет в разрезе характеристик, а перечень товаров (номера GTIN), для которых надо заказать этикетки с КиЗ регистрируется для каждой характеристики КиЗ.

Можно заказать неперсонифицированные КиЗ. В этом случае в карточке КиЗ информация о товарах (GTIN) не указывается. При маркировке в неперсонифицированный КиЗ добавляется информация о конкретном товаре.

Ввод информации о новом товаре, подлежащем маркировке

При вводе информации о новом товаре и выборе соответствующего вида номенклатуры будут автоматически заполнены все значения по умолчанию, в том числе и информация о классификаторе продукции. В качестве штрихкода регистрируется код GTIN, присвоенный данной продукции GS1.

Если для товара ведется учет по характеристикам, информация о GTIN (штрихкоде) регистрируется для каждой характеристики товара.
