Правила ценообразования

Группе магазинов или каждому магазину в отдельности может быть назначено свое правило ценообразования. Перечисленные в правиле виды цен образуют матрицу цен для продажи в розницу. С помощью видов цен поддерживается также разграничение управления ценами в разрезе номенклатурных групп и объектов сети – центрального офиса и магазинов.

Цены, назначаемые в офисе, передаются согласно правилам ценообразования в соответствующие магазины и, спустя некоторое время, обусловленное технологическим процессом предпродажной подготовки, могут быть применены и переданы на кассы ККМ магазина с помощью обмена РИБ по кассам.

Список правил открывается по ссылке Маркетинг – Ценообразование – Правила ценообразования. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные правила, задавая поиск по значениям различных реквизитов.

Новую запись в справочник можно добавить с помощью кнопки Создать в командной панели списка. Наименование правила ценообразования отражает область действия данного правила: конкретный магазин или группу магазинов, составленную, например, по формату магазинов или по региональной принадлежности.

Выбранный пользователем из списка основной Вид цен будет действовать для всех позиций номенклатуры. Пользователь может вывести Правило ценообразования из использования или отсрочить начало действия вновь созданного Правила ценообразования, присвоив ему Статус «Не действует».

Пример. Централизованное ценообразование. Розничные цены всей номенклатуры настраиваются и рассчитываются в центральном офисе и действуют для всех магазинов сети. Для настройки создается Вид цен «Розничные цены» и указываем его в правиле ценообразования в качестве основного вида цен. Табличная часть не заполняется.

Учет НДС в цене продажи.
Флажок Цена включает НДС определяет содержание НДС в действующих розничных ценах магазина. В зависимости от этого флага пользователю будет предложен для выбора список доступных видов цен, содержащих или не содержащих НДС.

Уточнение розничной цены по ценовым группам.

Если в системе включена настройка параметра "Ценовые группы" в Администрирование – Маркетинг, то становится доступна табличная часть Уточнение по ценовым группам, где пользователь может перечислить ценовые группы номенклатуры, на которые действуют другие правила ценообразования и указать для них соответствующие виды цен.

Пример 1. Децентрализованное ценообразование. Розничные цены на все позиции номенклатуры назначаются менеджером магазина и действуют соответственно в своем магазине.
- Для каждого магазина создается вид цен «Розничная цена магазина №…» или «Розничная цена региона…».
- Для каждого магазина создается Правило ценообразования «Розничные цены магазина № …», где в качестве основного вида цен указывается соответствующий Вид цен.

Пример 2.
Смешанное ценообразование. Розничные цены на часть ассортимента назначаются менеджером магазина, остальные – управляются в центральном офисе.
- Создается по крайней мере два вида цен, один из которых доступен менеджеру офиса «Розничная цена офиса», а другой «Розничная цена магазина №…» - менеджеру магазина.
- Создается ценовая группа номенклатуры «Номенклатура магазина», в которую включается номенклатура, управляемая в магазине.
- В Правиле ценообразования «Розничные цены магазина № …» в качестве основного Вида цен номенклатуры указываем «Розничную цену офиса». В табличной части указываем Ценовую группу «Номенклатура магазина» и соответствующую ей «Розничную цену магазина №…».
