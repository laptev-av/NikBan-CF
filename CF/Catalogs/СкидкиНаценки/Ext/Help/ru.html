Скидки (наценки, ограничение)

Для того чтобы система автоматически рассчитывала скидки при продаже товара, необходимо создать в системе элемент справочника Скидки (наценки, ограничения). В действие скидки вводятся документом Маркетинговая акция, в котором указываются магазины и периоды действия скидок, перечисленных в документе.

Список элементов справочника открывается по ссылке Маркетинг – Маркетинговые акции - Скидки (наценки, ограничение) в виде дерева. В узлах дерева пользователь может настроить правила совместного применения скидок, размещенных в подчиненных группах. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные элементы, задавая поиск по значениям различных реквизитов.

Группы скидок

На покупку могут действовать несколько скидок одновременно, поэтому необходимо установить правила совместного применения, то есть правила, по которым будет рассчитываться итоговая скидка. Перейдите в список Маркетинг – Маркетинговые акции - Скидки (наценки, ограничения), нажмите кнопку Создать новую группу и выберите вариант совместного применения для всех скидок, входящих в эту группу:

· вытеснение,

· минимум,

· максимум,

· сложение,

· последовательное применение.

Важно! В корневой группе программно установлено правило Сумма.

Алгоритм определения итоговой скидки для каждой позиции чека работает следующим образом:

1. По параметрам чека из всего множества действующих скидок определяются скидки, для которых выполнились условия.

2. Начиная с периферийных веток дерева, по правилам совместного применения последовательно для каждой вложенной группы определяется итоговый результат действия скидок.

Результат взаимодействия скидок в группе с правилами Сложение и Умножение определяется всеми скидками группы. При применении правила Сложение итоговая скидка – это сумма скидок. При использовании правила Умножение первой применяется верхняя в списке группы скидка, затем вторая скидка применяется к результату действия предыдущей скидки и т. д. Суммируются любые виды скидок. Например, одновременно покупатель может получить скидку 3 % на сумму чека по времени продажи и чайное ситечко в подарок при покупке чайника, если купит чайник до 11 утра. Пример последовательного применения скидок:сначала применяются скидки на товарные строки, процентные или суммой, с учетом этих скидок определяется сумма чека, к которой применяется процентная скидка по времени покупки.

В остальных случаях итоговая сумма скидки – одна из скидок группы. В случае применения правила Вытеснение верхняя в пределах группы скидка, для которой выполнились условия, безусловно вытесняет все расположенные ниже. Положением скидки в группе можно управлять с помощью стрелок в командной панели. При совместном применении скидок по правилам Максимум или Минимум применяется скидка, которая принимает соответственно максимальное или минимальное абсолютные значения суммы скидки.

Важно! Применение правил условного и безусловного вытеснения скидок зависит от сопоставимости вида и области предоставляемой скидки.

Новую запись в справочник можно добавить с помощью кнопки Создать в командной панели списка. Чтобы настроить скидку нужно указать:

· Наименование;

· Группа совместного использования;

· Тип скидки- описывает преимущества, которые получает покупатель при выполнении условий скидки;

· Размер предоставляемой скидки и ее значение;

· Область предоставления скидки - товары, на которые предоставляется скидка могут не совпадать с товарами, которые перечислены в условиях скидки.

· Условия предоставления скидки - условия, при выполнении которых предоставляется автоматическая скидка и может быть предоставлена ручная скидка;

· Взаимодействие скидок настраивается с помощью размещения скидок в определенных группах справочника.

Тип скидки

В системе поддерживаются следующие виды скидок:

· Процент – скидка составляет определенный процент от суммы в указанной области предоставления скидки;

· Процент от суммы предыдущей покупки – сумма скидки определяется в виде процента от суммы предыдущей покупки, зарегистрированной на данного покупателя (дисконтную карту);

· Сумма – абсолютное значение скидки в указанной области предоставления;

· Использовать вид цены – скидка заключается в изменении вида цены, используемой в документе;

· Выдать подарок – скидка представляет собой часть покупки или специально выделенные товары;

· Вывести сообщение – при выполнении условий на экран кассира выводится сообщение о предоставляемой скидке;

· Запретить розничную продажу – возможность ограничить продажи определенной группы товаров в указанное пользователем время;

· Начислить бонусные баллы в количестве – при выполнении перечисленных в скидке условий на бонусный счет, указанный в дисконтной карте покупателя, будет начислено определенное количество баллов;

· Начислить бонусные баллы процентом от суммы - при выполнении перечисленных в скидке условий на бонусный счет, указанный в дисконтной карте покупателя, будет начислено округленное до целого количество баллов, рассчитанное по указанному в скидке проценту.

· Внешняя обработка. Расширяет возможность видов предоставляемых скидок с помощью внешних обработок. Внешняя обработка загружается в справочник Дополнительные отчеты и обработки. При этом необходимо выбрать один из типов: скидка или условие скидки. В качестве примера с конфигурацией поставляются условие для скидки "На каждую N-ую строку чека", и обработка определения преимущества "Скидка на произвольный комплект".

Важно. При расчете скидки преимущество в виде суммы, подарка или определенного количества бонусов предоставится покупателю столько раз, сколько раз выполнится в чеке условие предоставления. Кратность предоставления может использоваться только для условий "За разовый объем продаж" с типом "не менее" и для комплектов покупки.

Скидка процентом или суммой

Нужно выбрать в поле Способ предоставления значение Процент или Cумма и указать в следующем поле величину скидки. Скидка в виде процента рассчитается от суммы документа или от суммы строк документа, указанных в Области предоставления скидки.

Перевод на другой вид цен при выполнении условий скидки

В поле Способ предоставления выберите значение Использовать вид цены и в следующем поле выберите из справочника нужный Вид цен. При выполнении условий скидки в определенных строках документа (или в целом в документе) цена будет установлена по указанному в скидке виду цен.

Скидка в виде подарка

Скидки в виде подарков покупателю учитываются двумя способами:

· Подарки предоставляются из числа выбранных покупателем товаров;

· Предоставляемые подарки перечислены в отдельном списке.

В первом случае нужно установить флаг Из корзины покупателя и в следующем поле указать то количество товаров с наименьшей ценой из документа, которые будут считаться подарками. Сумма по этим позициям будет вычтена из суммы чека и распределена по оставшимся позициям в виде скидки.

Пример. Скидка "Три диска по цене двух" настраивается следующим образом:
- Выбираем предоставляемую скидку "Подарок", устанавливаем флаг "Из корзины покупателя" и количество - 1.
- Устанавливаем условие скидки:"За разовый объем продаж": "Количество в документе не менее трех из сегмента номенклатуры "Диски"".

Во втором случае в скидке указывается перечень подарков и их количество. При выполнении условий скидки должны быть выданы все перечисленные подарки.

Пример. При покупке машины предоставляется комплект зимних шин на дисках:4 шины и 4 диска. В список подарков в этом случае нужно внести две номенклатурные строки - шины и диска и указать в обеих количество - 4шт.

Если в табличной части подарков установлен флаг Из списка, то предоставляется подарок, описанный только одной строкой списка.

Пример. В декабре при покупке канцелярских товаров на сумму более 1000руб. покупателю предлагается подарок на выбор:блокнот или карандаш.
В этом случае в список подарков также вносится две позиции номенклатуры - блокнот и карандаш, для обеих позиций указывается количество - 1 и устанавливаем флаг "Из списка". При выполнении условий скидки в режиме РМК кассиру предоставится выбор из двух позиций списка подарков.

Подарки, предоставляемые из списка, могут учитываться в бухгалтерском учете как маркетинговые материалы. В этом случае они не включаются в товарную часть чека, а списываются отдельным документом.
Если же подарки из списка учитываются как товары, то они могут быть учтены в составе проданного товара. Стоимость подарков в этом случае вычитается из суммы чека и распределяется в виде скидки по всем позициям чека. В этом случае нужно установить флаг Учитывать подарок как продажу.

Пример. Если блокноты и карандаши в предыдущем примере одновременно предоставляться как подарки и продаваться по установленной розничной цене или иметь фирменный логотип и распространяться как имиджевая реклама. В первом случае выданные подарки учитываются как продажи, во втором - как рекламные расходы.

Сообщение кассиру

Выберите в поле Способ предоставления значение Вывести сообщение и установите время появления сообщения в интерфейсе РМК: В момент расчета скидок или После оформления чека. Введите Текст сообщения.

Пример. На предприятии применяются дисконтные карты с разным оформлением. При превышении определенного порога накопления нужно произвести замену карты.
Настроим условие скидки: при достижении накоплений 10000руб. за весь период по дисконтной карте с видом "Серебряная" и установим вид предоставляемой скидки - "Сообщение" после оформления чека: "Накопление по карте "Серебряная" превысило установленный порог. Выдайте карту "Золотая". Получив сообщение кассир произведет замену карт.

Скидка - процент от суммы предыдущей покупки

Нужно выбрать в поле Способ предоставления значение Процент от суммы предыдущей покупки и указать в следующем поле процент скидки. Сумма предоставляемой скидки рассчитается процентом от суммы предыдущей покупки, в которой фиксировалась карта покупателя. Для данной скидки обязательным является условие предоставления - за накопленный объем продаж от суммы предыдущей покупки.

Запрет продажи алкоголя по времени

Нужно выбрать в поле Способ предоставления значение перечисления Запретить розничную продажу и указать в поле Номенклатура сегмента сегмент, содержащий алкогольную продукцию. Для данной скидки обязательным является условие предоставления - за время продажи, в котором необходимо обязательно указать временные периоды запрета продаж.

Товары, на которые должна быть предоставлена скидка

Товары, на которые предоставляется скидка составляют Способ предоставления скидки. Скидка может предоставляться либо в документе в целом, либо в строке, в которой выполнились все условия скидки, подобранные в табличной части из справочника условий предоставления скидки. Помимо этого, скидка может предоставляться только на определенные товары, перечисленные в сегменте номенклатуры. В сочетании с разнообразными условиями скидки можно достичь разного результата.

Пример. При покупке двух товаров из группы "Кофеварки" 2% скидка на сумму документа. Нужно выбрать условие "Количество в документе не менее двух по сегменту "Кофеварки"" и установить скидку 2 "Процента" "В документе".

Пример. При покупке двух одинаковых кофеварок из группы "Кофеварки" 2% на эти кофеварки. Нужно выбрать условие "Количество в строке не менее двух по сегменту "Кофеварки"" и установить скидку 2 "Процента" "В строке, в которой выполнилось условие".

Пример. При покупке двух кофеварок из группы "Кофеварки" 2% на вентиляторы. Нужно выбрать условие "Количество по документу не менее двух по сегменту "Кофеварки"" и установить скидку 2 "Процента" область предоставления "в документе" "На сегмент номенклатуры" "Вентиляторы".

Пример. При покупке двух кофеварок из группы "Кофеварки" 2% на вентиляторы, если их не меньше двух одного наименования.
Нужно выбрать условие "Количество в документе не менее двух по сегменту "Кофеварки"" и второе условие "Количество в строке не меньше двух по сегменту "Вентиляторы"".
Установить скидку 2 "Процента" область предоставления "В строке" "На сегмент номенклатуры" "Вентиляторы".

Настройка накопительной скидки

При вычислении суммы накоплений может учитываться сумма текущего чека. Для этого необходимо установить флажок Учитывать текущий чек в накоплениях в условиях соответствующей скидки.

Алгоритм расчета скидок

В части определения выполнения условий скидки для данного чека - все скидки обрабатывается последовательно, в соответствии с параметром Порядок применения:

· Для первой скидки первой подгруппы первой группы и т.д. по дереву скидок проверяется выполнение условий;

· Если условия выполнились, производится расчет величины скидки и, если необходимо, распределение по строкам чека;

· Далее осуществляется переход к следующей скидке в группе;

· После расчета всех скидок в группе в соответствии с правилом совместного применения производится вычисление итоговой скидки по группе;

· После этого осуществляется переход к следующей группе совместного применения.

Если условия скидки не выполняются, алгоритм пропускает скидку. После проверки и расчета скидок первой группы, алгоритм переходит к обработке второй группы нижнего уровня и так далее, пока не будут проверены все действующие скидки.

Поскольку условия применения скидки проверяются последовательно, появляется возможность учитывать величину уже рассчитанной скидки в процессе действия алгоритма. Если установить флаг Учитывать примененные скидки в условии скидки, то при проверке выполнения условий данной скидки будут учтены все уже рассчитанные до этого скидки.

Пример. В магазине действуют скидка на телевизоры 5% (порядок применения 1) и скидка на сумму чека более 25 000 руб. - 2% (порядок применения 2). Скидки размещены в группе с правилом "Последовательное применение".

При продаже телевизора стоимостью 26 300 руб. Расчет скидки будет следующим:

При установленном флаге "Учитывать примененные скидки". Скидка на телевизор составит 1315руб. При проверке условий скидки на чек будет учтена сумма чека 26300-1315 = 24985 руб., что меньше 25000 и, следовательно, скидка рассчитана не будет.

При снятом флаге "Учитывать примененные скидки" сумма скидки на телевизор будет прежней - 1315 руб., но при проверке условий для скидки на чек в расчет будет приниматься стоимость телевизора без учета скидки - 26300, что больше значения 25000, указанного в условии скидки. Скидка на чек рассчитается в размере 2% от стоимости телевизора с учетом скидки - 24985*0,02= 499,7 руб. Итоговая сумма чека составит 24485,3 руб., скидка - 1814,7руб. (6,9% от стоимости телевизора). Использование флага "Учитывать примененные скидки" в группах с последовательным применением скидки приводит к тому что база проверки условий применения и расчета скидки совпадают.

Если же скидки, указанные в примере, будут размещены в группе с правилом применения "Суммирование", то сумма скидки при установленном флаге "Учитывать примененные скидки" останется прежней - 1315 руб., а при снятом составит 1841 руб. (7% от стоимости телевизора). Сочетание таких правил начисления скидок также становится более прозрачным для покупателя.

Автоматический перерасчет цен

Если производительность кассового компьютера достаточно высокая, то можно воспользоваться возможностью перерасчета скидок по мере формирования документа. Для этого пользователю необходимо предоставить дополнительное право: Автоматический расчет скидок при редактировании документа продажи (Администрирование – Пользователи и права – Дополнительные права пользователей - Маркетинг).

Совместное применение скидок

Поддерживается совместное применение разнородных преимуществ, получаемых покупателем при расчете скидки. Преимущества оцениваются в стоимостном выражении:подаркам можно назначить цену продажи, сообщения кассиру имеют нулевую стоимость, бонусы пересчитываются по курсу, заданному в бонусной программе. Полученные значения скидок сравниваются для каждой товарной строки и применяются в соответствии с правилами совместного применения, указанными для группы скидок. При этом действует принцип неделимости подарков и сообщений:стоимость подарка или сообщений сравнивается с величиной другой скидки, рассчитанной на те же строки (или на весь чек), что и подарок.

Пример. В садовом центре действуют следующие скидки:

· Скидка "На садовый инструмент" в виде садового совка (подарок) стоимостью 125 руб. при покупке набора, состоящего из лопаты, грабель и вил.

· Скидка "На саженцы" - 10%

· Скидка "На чек" суммой более 3000 - 2%.

· Скидки размещены в группе "Максимум".

Покупатель выбрал товары для покупки

· Лопата Fiskars        - 2 990 руб

· Вилы Fiskars           - 1 320 руб.

· Грабли Fiskars        - 2 440 руб.

· Секатор Fiskars      -    750 руб.

· Яблоня "Штрифель" - 1 500 руб.
Сумма чека без скидки: 9 000 руб.

При определении предоставленных скидок будет рассчитана скидка "На чек" для позиций Лопата Fiskars, Вилы Fiskars и Грабли Fiskars - 0,02*(2990+1320+2440)= 135 руб., которая будет сравниваться по правилу "Максимум" со стоимостью садового совка - 125 руб. В результате скидка-подарок не будет применена. Скидка "На саженцы" вытеснит скидку "На чек" для позиции "Яблоня "Штрифель"" и для позиции "Секатор Fiskars" рассчитается скидка 2%.

В результате сумма скидки составит 135+150+15 = 300 руб. и сумма чека со скидкой составит 8 700 руб.

Ограничение максимальной суммы скидки

В программе предоставляется две возможности ограничения суммы скидки - максимальная сумма скидки, предоставляемая в данной группе скидок и по определенному для магазина виду цен - минимальной цене продажи.

В каждой группе совместного применения можно задать максимальный размер скидки в виде суммы или процента от суммы по строке без учета скидок. Если скидка, рассчитанная на данный момент для строки превысит величину ограничения, то по данной строке предоставлена скидка в размере ограничения. Для настройки ограничений в группе необходимо установить флаг Ограничение размера подчиненных скидок в форме группы скидок.

Пример. Для группы верхнего уровня установлено ограничение скидки 20%. После последовательного расчета и применения по установленным правилам всех действующих скидок будет произведена проверка на превышение 20%порога от исходной цены товара и в зависимости от результата либо останется прежняя сумма скидки, либо она будет уменьшена до 20%.

Затем, в случае, если в магазине установлен Вид минимальных цен, получившаяся цена со скидкой будет сравниваться с минимальной ценой. Если цена со скидкой окажется меньше, то минимальная цена станет ценой продажи.
