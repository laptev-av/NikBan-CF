Склады

В каждом магазине может быть неограниченное количество складов. Каждый склад принадлежит одной организации. В то же время в магазине может быть несколько складов, продажа товаров с которых осуществляется от имени одной организации. Это позволяет реализовать схему, когда собственниками магазина является несколько независимых юридических или физических лиц (организаций).

Список складов открывается НСИ - Предприятие - Склады. Для удобства работы Пользователь может отфильтровать список складов по магазину, выбрав нужное значение в поле Магазин в шапке списка. С помощью команды Ctrl+F можно отфильтровать список по любой комбинации символов. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные склады, задавая поиск по значениям различных реквизитов.

Новый склад можно добавить через кнопку Создать в командной панели списка Складов.

В открывшейся форме ввести наименование и выбрать тип склада: Складское помещение или Торговый зал.

1. Складское помещение – складское помещение, в котором хранятся товары (подсобка, основной склад магазина и т.д.). Этот склад является своеобразным буфером, с которого товары распределяются по торговым залам магазина. С этого же типа склада оформляются оптовые продажи товаров конкретному покупателю (юридическому лицу). С данного вида склада не предусмотрено оформление розничных продаж.

2. Торговый зал – складское помещение, с которого осуществляется розничная продажа товаров. В магазине может быть сколь угодно много складов этого типа. В качестве примера можно привести розничные отделы магазинов:бакалея, кондитерские изделия, винно-водочный отдел и т.д.

Примечание. Кроме этих типов складов в программе предусмотрен еще один тип склада – склад центрального офиса. Склад центрального офиса – это тот склад, с которого будут отгружаться товары из центрального офиса. При начале работы информацию о таких складах заполнять не надо. Информация об этих складах заполняется в момент «объединения баз» или, если базы уже объединены, при вводе нового склада в управляющей системе, не являющегося магазином для текущего центрального узла конфигурации «Управление розничной торговлей».

Склады с типом Торговый зал могут создаваться в целях обособления учета товародвижения. Например, физически торговый зал может быть представлен единым помещением, но в нем выделено несколько зон учета (секции), представленных в конфигурации в качестве складов.

Нужно указать организацию, от имени которой по умолчанию будут оформляться документы. Для склада с типом Торговый зал этот реквизит обязателен. В соответствии с этим реквизитом и распределением продаж по складам магазина производится автоматическое формирование чеков продажи при использовании РМК. Подробнее см. «Оформление розничных продаж и возвратов в интерфейсе РМК»

Если продажи со склада облагаются единым налогом на вмененный доход, то нужно создать запись в списке Применения ЕНВД для данного склада. При этом дата автоматически заполнится первым днем текущего месяца и в поле организации – значением организации склада.

Если по каким-то причинам действие ЕНВД на данном складе отменено, нужно создать новую запись с помощью копирования записи введения ЕНВД на данном складе и, установив дату планируемого окончания действия ЕНВД, снять флажок Розничная торговля облагается ЕНВД.
