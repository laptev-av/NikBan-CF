Свертка информационной базы

В программе предоставлена возможность уменьшения объема баз данных магазинов сети и офиса.

Уменьшение базы данных заключается в удалении всех документов движения денежных средств, товаров и ценообразования до указанной пользователем даты и формировании документов ввода остатков. Затем записи справочников, которые не упоминаются в оставшихся документах также могут быть удалены с помощью специального сервиса. При определении даты свертки необходимо руководствоваться соображениями не только параметрами быстродействия программы, но и требованиями оперативного и регламентного учета. Не следует устанавливать дату свертки в открытом регламентном периоде, т.к. может возникнуть желание уточнить те или иные данные по документу. Период хранения документов может быть обусловлен также установленным на предприятии порядком ведения взаиморасчетов с поставщиками, периодом статистики для составления заказов, программами управления лояльностью покупателей. В некоторых случаях, например, при выраженной сезонной зависимости продаж товаров удобно иметь на компьютере менеджера две или даже несколько рабочих баз: одна для обслуживания текущих операций, остальные - для анализа прошлогодних продаж.

В сворачиваемом периоде должны быть завершены все операции с товаром и денежными средствами: проведены все документы, полностью завершены инвентаризации, закрыты перемещения товаров по сети и передачи денежных средств из кассы в кассу. При использовании РИБ завершены все обмены данных.

При большом размере сети или при наличии в сети магазинов большого формата, рекомендуется проводить сокращение объема баз в два этапа - сначала обработать периферийные базы, затем базу центрального узла. Для уменьшения объема периферийных баз магазинов необходимо установить параметр Дата свертки в настройке синхронизации узла данного магазина или группы магазинов. При отработке регламентного задания обмена данными автоматически сформируется документ Ввод начальных остатков узла, отражающий остатки всех магазинов данного узла обмена. При совершении обмена все документы данного узла ранее даты свертки будут удалены.

После завершения процесса сокращения периферийных баз можно приступить к сокращению центральной базы. При свертке данных центрального узла рекомендуется следующий порядок действий:

· Создать резервную копию информационной базы

· Выполнить свертку данных

· Проверить результаты свертки (с помощью прикладных отчетов)

· Выполнить удаление данных в свернутом периоде

· Закрыть свернутый период для изменений:установить дату запрета изменения данных, равную дате свертки.

Свертка данных центрального узла производится с помощью сервиса Свертка информационной базы (Администрирование – Обслуживание – Регламентные операции) пользователем, имеющим административные права. Помощник проведения свертки предоставляет возможность на первом шаге создать резервную копию базы. На следующем шаге проверит наличие узлов РИБ и предложит пользователю выполнить синхронизацию данных.

На следующей странице помощника Администратор может задать параметры проведения свертки.

Пользователь указывает дату свертки центрального узла, которая не может быть позже даты свертки какого-либо из узлов РИБ, и выбирает один из двух режимов работы сервиса:

· Автоматический режим. Все этапы свертки будут выполняться последовательно за один прием.

· Контролируемый режим. Каждый этап свертки будет выполняться обособленно. В этом режиме можно остановить процесс свертки после выполнения одного из этапов, и закрыть форму помощника. При следующем открытии формы процесс будет продолжен с того же места.

На следующем шаге помощник проверит информационную базу на наличие незавершенных операций и выведет диагностику готовности базы к свертке.

Если при работе процедур обработки возникли проблемные ситуации и процесс прервался, необходимо снова открыть помощник проведения свертки. В открывшейся форме пользователь сможет просмотреть Журнал регистрации и решить, может ли процесс свертки быть продолжен или необходимо восстановить информационную базу из резервной копии. Если свертка проводится первый раз рекомендуется провести ее на копии базы. При успешном завершении - выполнить процесс на рабочей базе.

Этапы свертки данных:

1. Формирование документов ввода остатков.

2. Удаление движений регистров, пометка на удаление документов;

3. Проведение документов ввода остатков.

Формирование документов ввода остатков. На этом этапе рассчитываются остатки на начало дня указанной пользователем даты и формируются следующие документы:

· "Ввод начальных остатков узла" для текущего (центрального) узла;

· "Изменение ассортимента" для каждой из операций документа;

· "Ввод остатков квоты ассортимента", актуальные на дату свертки для каждого формата магазина;

· "Установка себестоимости" формируется в зависимости от настроек расчета себестоимости - либо в целом по сети, либо в разрезе магазинов.

· "Ввод остатков по накопительным скидкам",

· "Ввод остатков расчетов с поставщиками",

· "Начисление и списание бонусных баллов".

Пользователь может просмотреть сформированные документы ввода остатков.

На втором этапе устанавливаются пометки на удаление всех документов, вошедших в период свертки, и удаляются записи регистров, сформированные этими документами.

Наконец, на третьем этапе проводятся сформированные документы ввода остатков. После завершения этапа результат можно проверить с помощью отчетов.

После проверки корректности введенных остатков, пользователь может выявить и удалить освободившиеся элементы справочников, движения по которым удалились из регистров, с помощью сервиса Удаление помеченных объектов. После проведения обмена данными по РИБ, сведения об удаленных элементах справочников распространятся по сети.

Также рекомендуется установить дату запрета изменения данных - не ранее даты свертки.

Механизм свертки имеет ограничения. Некоторые данные не могут быть свернуты:

· Движения серий товаров,

· Данные о подсчете посетителей магазина,

· История изменения себестоимости.

· Эти сведения при свертке будут удалены.
