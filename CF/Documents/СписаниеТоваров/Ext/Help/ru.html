Списание товаров

Документ Списание товаров служит для регистрации убыли товаров в результате порчи, боя или в связи с обнаруженной при инвентаризации склада недостачей.

Обычно на предприятии устанавливаются определенные правила списания товаров, включающие в себя организационный регламент списания, направления списания, бюджеты списания по направлениям. Например, может быть установлено, что списание пришедших в негодность товаров производится комиссией определенного состава с периодичностью месяц или неделя. При проведении инвентаризаций пересчет товаров часто совмещается с выявлением недоброкачественного товара и проведением его списания.

Список документов открывается по ссылке Склад – Складские операции – Списание товаров. Для удобства работы Пользователь может установить отбор по полям Магазин, Склад. С помощью сервиса Поиск(Ctrl+F), который запускается из командной панели списка, Пользователь может выбрать нужные документы, задавая поиск по значениям различных реквизитов. Выбранный документ может быть напечатан по форме Акт списания товаров и регламентированную форму акта о списании, ТОРГ-16. Список товаров без указания цен и сумм можно также просмотреть в печатной форме Опись номенклатуры.

Новый документ можно добавить с помощью кнопки Создать в командной панели списка. Если отражение излишков и недостач в магазине проводится по ордерной схеме, то списание можно оформить на основании документов Ордер на отражение недостач товаров и Ордер на отражение результатов пересчетов товаров. Кроме того, документ Списание товаров можно создать на основании документа Приказ на пересчет товаров, воспользовавшись одним из двух следующих способов:

1 способ. По ссылке Склад – Инвентаризация – Приказы на перерасчет - Создать на основании выбрать Списание товаров. При этом табличная часть документа будет заполнена всеми товарами, по которым при пересчете по данному приказу была выявлена недостача и эта недостача не была закрыта зачетом пересортицы.

В конфигурации «Розница» для регистрации направлений списания используется справочник Аналитика хозяйственной операции, в котором для операции списания могут быть созданы несколько аналитик. Например, списание на оформление витрин, списание подарков, предоставленных в качестве скидок, списание брака, списание недостачи на материально ответственных лиц, естественная убыль продовольственных товаров, списание по инвентаризации. В системе предусмотрены предустановленные признаки списания:

· Списание по инвентаризации – документ создается при проведении полных или частичных инвентаризаций;

· Списание на затраты – например, списание товаров на оформление витрин магазина;

· Списание на затраты (подарки) – товарные позиции, предоставленные покупателю в качестве подарка во время проведения маркетинговых акций.

2 способ. Для вызова помощника заполнения причин списания в окне документа Приказ на пересчет товаров нужно в выпадающем меню Сервис выбрать Заполнить причины списания и в открывшемся окне назначить необходимые значения поля Аналитика хозяйственной операции.

Выбранную аналитику можно записать во все строки табличной части, где аналитика ещё не заполнена, с помощью кнопки Заполнить пустые строки. Очистить значения аналитик во всех строках табличной части можно с помощью кнопки Очистить все строки. Кроме того, значение аналитики можно изменить отдельно в каждой строке.

Работа помощника может быть завершена с помощью кнопки Создать документы. В результате будет создано столько документов Списание товаров, сколько различных значений аналитик Пользователь выбрал для списываемых товаров в процессе работы с помощником. В каждом созданном документе табличная часть автоматически заполняется товарными позициями с соответствующими причинами списания.

Заполнение шапки документа Списание товаров

Если документ был создан на основании приказа на пересчет товаров, ордера на отражение результатов пересчета или ордера на отражение недостач, то поля шапки заполняются автоматически значениями полей документа-основания. Если документ был создан с использованием помощника заполнения причин списания, то поле Аналитика хозяйственной операции, в котором зафиксирована причина списания, тоже заполняется автоматически.

При создании документа Списание товаров вручную некоторые поля шапки документа могут быть автоматически заполнены значениями, установленными для пользователя по умолчанию, или теми значениями, которые были выбраны пользователем для быстрого отбора в списке документов. Пользователь редактирует или заполняет следующие поля:

· Магазин, в котором производится списание;

· Склад выбранного магазина, на котором производится списание;

· Организация, товары которой списываются;

· Аналитика хозяйственной операции - причина списания.

Для регистрации причин списания используется справочник Аналитики хозяйственных операций, в котором для хозяйственных операций Списание на затраты и Списание по инвентаризации могут быть созданы несколько аналитик. Справочник Аналитики хозяйственных операций заполняет Администратор на странице Запасыи Закупки раздела Администрирование. Если документ Списание товаров был создан на основании приказа на пересчет товаров или ордера на отражение результатов пересчета, то Пользователь может выбрать те аналитики, которые соответствуют хозяйственной операции Списание по инвентаризации. Иначе Пользователь выбирает причину списания среди аналитик, соответствующих хозяйственной операции Списание на затраты.

Заполнение табличной части

Если документ был создан на основании ордера, то табличная часть заполняется из табличной части ордера-основания. Если документ был создан на основании приказа на пересчет товаров, то табличная часть будет заполнена всеми товарами, по которым при пересчете по данному приказу была выявлена недостача и эта недостача не была закрыта зачетом пересортицы. Если документ был создан с использованием помощника заполнения причин списания, то табличная часть будет заполнена товарами, которые Пользователь отобрал в процессе работы с помощником.

Добавить товары в табличную часть можно следующими способами:

· с помощью кнопки Добавить командной панели табличной части документа;

· с помощью сервиса Подобрать товары, который запускается из командной панели табличной части;

· загрузить список товаров из терминала сбора данных, в этом случае, если табличную часть предварительно не очистить, то сведения по количеству товара из ТСД просуммируются с количеством в соответствующих товарных строках табличной части;

· по штрихкоду. Добавление товара в табличную часть по штрихкоду. Чтобы добавить товар по штриховому коду, нужно перейти в табличную часть и считать штриховой код сканером. Кроме того, можно нажать кнопку Поиск по штрихкоду в командной панели или горячую клавишу F7 и ввести символы штрихкода вручную или с помощью сканера в открывшемся окне. Если данный штрихкод зарегистрирован в системе, то соответствующий ему товар будет добавлен в табличную часть. В противном случае в правой части формы появится сообщение об ошибке. Если товар с таким штрихкодом уже введен в табличную часть, то строка с данным товаром будет выделена, а количество увеличится на 1.

Ввод серий списываемых товаров

Серию товара необходимо вводить в строке табличной части документа Списание товаров при выполнении нескольких условий:на предприятии ведется учет товара по сериям, для данного вида номенклатуры в данном магазине настроена политика учета серий, согласно этой политике нужно учитывать серии при отражении недостач; схема списания товаров для данного магазина однофазная (без оформления ордеров).
В этом случае в строке товара в поле Количество будет отображен значок Статусы серии.
Значок красного цвета означает, что серии еще не введены. Для ввода серии нужно из поля Количество открыть форму Ввод количества и серий товаров и заполнить её. После корректного ввода серии и количества, значок серии меняет свой цвет на зеленый.

Для списания нумерованных подарочных сертификатов с истекшим периодом действия нужно воспользоваться сервисом Ввод подарочных сертификатов, который запускается из поля Количество, если в поле Номенклатура выбран подарочный сертификат.

Пользователь может указать цены списания товара вручную или с помощью следующих сервисов, расположенных в меню Цены командной панели табличной части:

· Заполнить цены по розничным ценам - цены будут заполнены значениями действующих розничных цен данного магазина;

· Заполнить цены по себестоимости - в зависимости от настроек будут указаны либо цены последней поставки, либо себестоимость номенклатуры, полученная из управляющей системы;

· Заполнить цены по виду цен - цены будут заполнены значениями того вида цен, который Пользователь выберет из списка доступных ему видов цен.
