Пересчеты товаров

Документ Пересчет товаров предназначен для фиксирования результатов проведенной на складе инвентаризации.

Документ Пересчет товаров не изменяет учетное количество товаров. Если в результате пересчета товаров выявлены излишки или недостачи, то для того, чтобы скорректировать данные управленческого учета, нужно на основании исходного документа Приказ на пересчет товаров создать документы Пересортица товаров, Списание товаров, Оприходование товаров. Кроме того, если в магазине принята ордерная схема оформления инвентаризации, то для того, чтобы привести учетное количество к фактическому нужно на основании исходного документа Приказ на пересчет товаров создать документ(ы) Ордер на отражение результатов пересчетов товаров.

Список документов пересчета товаров можно открыть по ссылке Склад - Инвентаризация - Пересчеты товаров.

Для удобства работы Пользователь может отфильтровать список по магазину, складу, приказу выбрав нужные значения в соответствующих полях шапки списка. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные строки, задавая поиск по значениям различных реквизитов.

Печатные формы документа

Для проведения ручного пересчета товаров можно из меню Печать распечатать список товаров для пересчета. Форма Инвентаризация и регламентная форма ИНВ-3 Инвентаризационная опись товаров содержат данные об учетном количестве товара. В форме Инвентаризация (пустая) и в регламентной форме ИНВ-3 Инвентаризационная опись товаров (пустая) данные об учетном количестве товара отсутствуют. Выбор формы зависит от принятой в магазине технологии учета.

После окончания пересчета и ввода фактического количества из меню Печать можно распечатать регламентную форму ИНВ-19 Сличительная ведомость.

Новый документ Пересчет товаров может быть создан только на основании документа Приказ на пересчет товаров. На основании одного приказа может быть создано несколько документов пересчета.

Заполнение шапки документа

При создании документа Пересчет товаров поля Магазин, Склад и Организация заполняются значениями, указанными в приказе-основании. В поле Основание доступна ссылка на приказ-основание. Кроме того, в шапке документа Пересчет товаров выводится текстовая строка с описанием параметров правила отбора товаров, которое было выбрано в приказе-основании.

Поле Статус заполняется пользователем вручную и может принимать следующие значения:

· В работе – документ создан, начался процесс физического пересчета товара учетчиками;

· Внесение результатов – вносятся и уточняются данные пересчета товаров;

· Выполнено – результаты приняты, в уточнении не нуждаются.

Заполнение табличной части перед пересчетом

Табличная часть документа Пересчет товаров при создании документа автоматически заполняется товарами и их текущими учетными остатками по складу, который указан в приказе-основании, и в соответствии с правилом отбора, которое было выбрано в этом приказе.

Список товаров для данного документа пересчета можно ограничить с помощью дополнительного отбора товаров. Для этого нужно задать параметры в поле Отбор и выполнить команду Заполнить с учетом отбора из меню Заполнить командной панели табличной части.

Добавить товары в табличную часть можно следующими способами:

· с помощью кнопки Добавить командной панели табличной части документа;

· с помощью сервиса Подобрать товары, который запускается из списка Заполнить в командной панели табличной части;

· по штрихкоду. Чтобы добавить товар по штриховому коду, нужно перейти в табличную часть и считать штриховой код сканером. Кроме того, можно нажать кнопку Поиск по штрихкоду в командной панели или горячую клавишу F7 и ввести символы штрихкода вручную или с помощью сканера в открывшемся окне. Если данный штрихкод зарегистрирован в системе, то соответствующий ему товар будет добавлен в табличную часть. В противном случае в правой части формы появится сообщение об ошибке. Если товар с таким штрихкодом уже введен в табличную часть, то строка с данным товаром будет выделена, а количество увеличится на 1.

Пользователь может указать учетные цены вручную или с помощью следующих сервисов, расположенных в меню Цены командной панели табличной части:

· заполнить цены по розничным ценам,

· заполнить цены по себестоимости,

· заполнить цены по виду цен.

Ввод результатов пересчета товаров

Фактическое количество товаров может быть заполнено учетными значениями с помощью команды Заполнить фактическое количество по учетному из меню Заполнить командной панели табличной части.

Ввод результатов пересчета товаров может производиться вручную. Для этого достаточно изменить фактическое количество по позициям с отклонениями.

Фактические данные также могут быть получены из терминала сбора данных с помощью кнопки Загрузить данные из ТСД командной панели табличной части.

После ввода фактических данных в начале строки с выявленными излишками появится пиктограмма «Плюс» зеленого цвета, а в строках с недостачами – «Минус» красного цвета. Строки с отклонениями можно отфильтровать из общего списка, установив в шапке документа флажок Только расхождения.

Итоговые данные пересчета

В итоговой части документа пользователю предоставляется информация об учетной и фактической сумме инвентаризации, общей сумме выявленного расхождения, а также в разбивке по излишкам и недостаче.
