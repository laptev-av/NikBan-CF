Эквайринговые операции

Это расчеты, произведенные покупателем с помощью платежной карты за отгрузки мелким оптом.

Список документов открывается по ссылке Финансы – Денежные средства - Эквайринговые операции. Для удобства работы Пользователь может установить отбор по полям магазин, организация и терминал – эти значения будут установлены при создании документа. С помощью команды Ctrl+F можно отфильтровать список по любой комбинации символов. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные строки, задавая поиск по значениям различных реквизитов.

Новый документ можно ввести с помощью кнопки Создать, выбрав одну из двух доступных операций:

· Поступление оплаты от покупателя

· Возврат оплаты покупателю

· Поступление аванса при оплате банковской картой

· Зачет аванса при передаче товаров (см. Приходный кассовый ордер)

Поступление оплаты от покупателя

Во вновь созданном документе нужно заполнить Магазин, выбрать Эквайринговый терминал из числа установленных в магазине. Банк-эквайрер определятся автоматически. Пользователь выбирает один из доступных Вид оплаты и вводит Номер карты выбранного из справочника Контрагента.

Сумма оплаты вводится в поле шапки документа с указанием Статьи движения денежных средств. Комиссия банка рассчитывается автоматически согласно условиям тарифа.

В табличной части можно также указать один или несколько документов, которые необходимо оплатить. Сумма к оплате рассчитается автоматически по суммам документов.

При проведении документа уменьшается задолженность покупателя на зафиксированную сумму оплаты и на эту же сумму увеличивается задолженность банка-эквайрера. Все взаиморасчеты с банком-эквайрером осуществляются в управляющей системе.

Возврат оплаты покупателю

Документы возврата платежа на платежную карту создаются на основании документов "Возврат товара от покупателя". В этом случае данные о магазине, организации и контрагенте, а также сумма к оплате заполняются из документа-основания. Пользователь корректирует по необходимости статью ДДС, вводит вид оплаты и номер карты покупателя и производит оплату, нажав на кнопку "Оплатить картой".

При выполнении операции проверяется возможность печати слип-чека на подключенном эквайринговом терминале. Если функция печати слип-чека не предусмотрена и не указан подключенный фискальный регистратор или АСПД будет выдано предупреждение о невозможности проведения операции.

После успешного завершения банковской операции появляется информационная надпись Банковская операция произведена и пробивается чек ККМ. В случае, если пробить чек на подключенном ФР или АСПД по каким-то причинам не удалось на форме появляется дополнительная кнопка "Пробить чек", с помощью которой пользователь может завершить оформление чека после устранения неисправности на устройстве.

Поступление аванса при оплате банковской картой

Поступление аванса при оплате банковской картой можно оформить с помощью документа Эквайринговая операция (Финансы - Денежные средства - Эквайринговые операции).

Документ оформляется аналогично ПКО. После выполнения команды Оплатить картой становится доступной команда Пробить чек, с помощью которой можно пробить чек на полученный аванс.

Можно также оформить возврат аванса с помощью Расходного кассового ордера или Эквайринговой операции с операцией Возврат оплаты покупателю.
