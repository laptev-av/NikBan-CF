Заказы поставщикам

Документ Заказ поставщику предназначен для фиксации согласованного с поставщиком заказа товаров. Кроме списка товаров в документе фиксируются согласованные цены, срок и место поставки заказанных товаров.

Список документов можно открыть по ссылке Закупки - Закупки - Заказы поставщикам. Для удобства работы Пользователь может отфильтровать список заказов по поставщику, магазину, складу, организации выбрав нужные значения в соответствующих полях шапки списка. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, Пользователь может выбрать нужные заказы, задавая поиск по значениям различных реквизитов. На основании документа Заказ поставщику можно оформить следующие документы:

· Поступление товаров – заполняется товарами из документа-основания;

· Приходный ордер – заполняется товарами из документа-основания;

· Расходный кассовый ордер – заполняется суммой наличных оплат, которые запланированы на дату создания ордера;

· Регистрация безналичной оплаты – заполняется суммой безналичных оплат, которые запланированы на дату создания документа;

· Заказ поставщику – реквизиты нового заказа заполняются по заказу-основанию, а табличная часть - недопоставленными товарами.

С помощью команды Печать сформированный заказ можно распечатать и передать поставщику, например, по факсу.

Новый документ Заказ поставщику можно сформировать вручную с помощью кнопки Создать в командной панели списка документов. При заказе нового товара предварительно создается карточка номенклатуры, в которой указываются свойства товара. При повторных заказах товара документ Заказ поставщику можно сформировать с помощью сервиса Анализ продаж и формирование заказов, который запускается из панели действий раздела Запасы.

Заполнение шапки документа

В шапке нового документа заполняются следующие поля:

· Магазин, в который планируется доставка товара;

· Склад, на который будет приходоваться товар, по умолчанию - склад поступления, указанный для данного магазина;

· Организация, по умолчанию подставляется организация склада;

· Поставщик выбирается из списка контрагентов;

· Дата заказа - дата оформления заказа, по умолчанию - текущая дата;

· Дата поставки, согласованная с поставщиком;

· Допустимое количество дней просрочки поступления заказа.

Некоторые из этих полей заполняются автоматически, если перед созданием нового заказа в списке заказов были установлены значения отбора или если новый заказ был создан с помощью сервиса Анализ продаж и формирование заказов.

Заполнение табличной части документа

Табличная часть документа может быть заполнена автоматически по результатам выполнения сервиса Анализ продаж и формирование заказов.

Добавить товары в табличную часть можно следующими способами:

· с помощью кнопки Добавить командной панели табличной части документа;

· с помощью сервиса Подобрать товары;

· по штрихкоду. Чтобы добавить товар по штриховому коду, нужно перейти в табличную часть и считать штриховой код сканером. Кроме того, можно нажать кнопку Поиск по штрихкоду в командной панели или горячую клавишу F7 и ввести символы штрихкода вручную или с помощью сканера в открывшемся окне. Если данный штрихкод зарегистрирован в системе, то соответствующий ему товар будет добавлен в табличную часть. В противном случае в правой части формы появится сообщение об ошибке. Если товар с таким штрихкодом уже введен в табличную часть, то строка с данным товаром будет выделена, а количество увеличится на 1.

В строке табличной части Пользователь может ввести или изменить Количество и Цену заказываемого товара.

Установить цены можно с помощью сервиса Заполнить цены по ценам прошлой поставки, который запускается из списка Заполнить цены в командной панели табличной части.

Сумма заказа

Сумма заказа рассчитывается в соответствии с настройками учета НДС на странице Дополнительно. При установленных флагах Учитывать НДС и Цена включает НДС сумма документа будет содержать рассчитанную по указанным в карточке номенклатуры ставкам сумму НДС.

Планирование оплаты заказанного товара

Долг поставщика на текущий момент отображается в группе Взаиморасчеты итоговой области документа. Долг поставщика складывается из сумм предоплат, произведенных до момента создания документа Заказ поставщику, и сумм недопоставленных оплаченных товаров.
Зачет предоплаты можно произвести из списка документов оплаты, который открывается по команде Зачет оплаты из командной панели документа Заказ поставщику.

Пользователь может зарегистрировать планируемые платежи по Заказу поставщику на странице Этапы оплат. По умолчанию система предлагает оплату наличными 100% в день планируемой поставки. Пользователь может отредактировать платежи, вводя нужное количество этапов, форму оплаты, долю суммы заказа в этапе. Для каждой записи он также может изменить дату платежа или количество дней отсрочки.

Возможно, автозаполнение этапов оплат в документах Заказ поставщику и Поступление товаров по заранее созданному графику оплаты. Заполнить этапы оплаты можно выбрав любой заранее созданный график оплаты из списка или предустановленный для поставщика (закладка Этапы оплат). Заполнение этапов оплаты производится во время сохранения или проведения документа.

Взаимозачет по недопоставкам можно произвести, нажав кнопку Подобрать взаимозачет командной панели табличной части. В открывшейся форме выводится список документов данного поставщика с указанием суммы, которую можно зачесть по каждому документу. По команде Перенести в документ будут сформированы строки этапов оплат, соответствующие выбранным документам.

Закрытие заказа

Информация о заказанной номенклатуре используется при анализе продаж и определении потребности в товарах. Актуальность данной информации поддерживается с помощью закрытия заказов, которое может производиться в двух случаях: при поступлении товара или отмене заказа поставщику.

Поддерживаемая системой технология не подразумевает частичные поставки по заказу. Предполагается, что по каждому документу Заказ поставщику будет зарегистрировано только одно поступление товаров в установленный менеджером по заказам период. Такой подход предоставляет сразу несколько преимуществ:

· позволяет планировать нагрузку отдела приемки магазина;

· позволяет учитывать уровень сервиса поставщика при исполнении заказов;

· позволяет закрывать просроченные заказы в автоматическом режиме;

· позволяет снизить объем работ менеджера по контролю невыполненных заказов.

Если все-таки планируется допоставка по частично выполненному заказу, то рекомендуется создать новый заказ на недопоставленную номенклатуру. Для этого нужно после оформления документа Поступление товаров перейти по ссылке в документ-основание Заказ поставщику и в меню Создать на основании выбрать пункт Заказ поставщику. При этом реквизиты нового заказа заполнятся по заказу-основанию, а табличная часть – недопоставленными товарами. Пользователю нужно лишь указать новую дату поставки.

Отмену заказа поставщику рекомендуется производить и в случае, если поставщик не смог привезти товар в указанный срок. В этом случае нужно, закрыв заказ, сделать перерасчет потребности и сформировать новый заказ уже с учетом текущих продаж и отмененной поставки.

При отмене (закрытии) заказа поставщику вручную нужно установить флаг Закрыт в шапке документа.

Автоматическое закрытие заказа

Автоматическое закрытие заказа производится в следующих случаях:

· при проведении документа Поступление товаров на основании документа Заказ поставщику;

· в результате выполнения регламентного задания Закрытие заказов поставщикам.

Регламентное задание анализирует незакрытые заказы в базе данных магазина, сравнивает текущую дату и дату поставки с учетом допустимого количества дней просрочки. В просроченных документах Заказ поставщику устанавливается флажок Закрыт. Регламентное задание настраивается на странице регламентных заданий, которую можно открыть по ссылке Администрирование - Обслуживание – Регламентные операции - Регламентные и фоновые задания. Рекомендуется настраивать данное регламентное задание на утренние часы, до начала работы менеджера по заказу товара.

Если установленное по умолчанию значение количества дней просрочки («ноль») не было изменено менеджером, то такой заказ не обрабатывается автоматической обработкой закрытия заказа. Его закрытие управляется менеджером вручную.

Оформление заказа эмитенту (поставщику) на изготовление КИЗ

Оформление заказа эмитенту (поставщику) не является обязательным. Однако использование этих документов позволит правильно отследить правильность оформления оплаты по заказам конкретных КиЗ.

Заказы на персонифицированные и не персонифицированные КиЗ необходимо оформлять раздельно.

В документе указывается количество этикеток (КиЗов) и цена за одну этикетку. Непосредственно из документа с помощью гиперссылки можно создать заявку на выпуск КиЗ.
