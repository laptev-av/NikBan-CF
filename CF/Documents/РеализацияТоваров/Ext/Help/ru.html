Реализации товаров

Документ Реализация товаров предназначен для оформления продажи товаров мелким оптом.

В зависимости от настроек магазина могут применяться две схемы оформления реализации:

· Однофазная – оператор отгружает товар и оформляет документ Реализация товаров, согласно которому уменьшаются остатки товара на складе магазина;

· Двухфазная (ордерная) – оператор вводит в систему документ Реализация товаров, в котором регистрирует информацию по ценам и количеству проданного товара, но фактически отгруженное со склада количество товара регистрирует кладовщик с помощью документа Расходный ордер.

Список документов можно открыть по ссылке Продажи - Прочее - Реализации товаров. Для удобства работы Пользователь может установить отбор по полям Магазин, Склад и Контрагент в шапке списка документов. С помощью команды Ctrl+F можно отфильтровать список по любой комбинации символов. С помощью сервиса Расширенный поиск (Alt+F), который запускается из командной панели списка, пользователь может выбрать нужные документы, задавая поиск по значениям различных реквизитов. Для документа предусмотрены следующие печатные формы:

• Расходная накладная,

• Опись номенклатуры (список товаров без указания цен и сумм),

• ТОРГ-12 (Товарная накладная),

• ТОРГ-12 (Товарная накладная без услуг),

• Акт об оказании услуг,

• Накладная М15,

• 1-Т (Товарно-транспортная накладная),

• Счет-фактура,

• УПД (Универсальный передаточный документ).

• Транспортная накладная,

Новый документ можно добавить с помощью кнопки Создать в командной панели списка.

Заполнение шапки документа

По умолчанию поля шапки заполняются в соответствии с настройками Пользователя, а если они не установлены, то значениями быстрого отбора в списке документов. Если соответствующий справочник содержит единственный элемент, то он будет подставлен по умолчанию.

Документ Реализация товаров оформляется с определенного склада хранения товаров. Поля Магазин, Склад, Организация и Контрагент должны быть заполнены обязательно. Организация, как правило, должна соответствовать складу. Иначе, для правильного учета остатков товаров в разрезе организаций, впоследствии будет нужно оформить документ Передача товаров между организациями.

Если на предприятии используются дисконтные карты, то Пользователь может ввести в поле Дисконтная карта номер карты, выбрать его из списка или считать штриховой (магнитный) код карты на подключенном к рабочему месту оборудовании. Если для карты определен владелец, то он запишется в поле Контрагент автоматически.

Для регистрации личных продаж сотрудников магазина продавец может быть указан как в шапке документа, так и в каждой строке.

Комиссионная торговля

При использовании комиссионной торговли, команда "Расшифровать товары по поставщикам", позволяет увидеть таблицу распределения товара (по поставщикам) при продаже.

Заполнение табличной части

Добавить товары в табличную часть можно следующими способами:

с помощью кнопки Добавить командной панели табличной части документа;

с помощью сервиса Подобрать товары;

загрузить из терминала сбора данных;

по штрихкоду. Чтобы добавить товар по штриховому коду, нужно перейти в табличную часть и считать штриховой код сканером. Кроме того, можно нажать кнопку Поиск по штрихкоду в командной панели или горячую клавишу F7 и ввести символы штрихкода вручную или с помощью сканера в открывшемся окне. Если данный штрихкод зарегистрирован в системе, то соответствующий ему товар будет добавлен в табличную часть. В противном случае в правой части формы появится сообщение об ошибке. Если товар с таким штрихкодом уже введен в табличную часть, то строка с данным товаром будет выделена, а количество увеличится на 1.

При подборе товаров Пользователь вводит их количество. Пользователь также может вручную изменить количество в строке табличной части. Если для отобранного товара ведется учет по характеристикам, то Пользователь при подборе выбирает соответствующие характеристики и может вручную отредактировать их в строке табличной части.

Серию товара необходимо вводить в строке табличной части документа Реализация товаров при выполнении нескольких условий: на предприятии ведется учет товара по сериям, для данного вида номенклатуры и для данного магазина настроена политика учета серий, согласно этой политике нужно учитывать серии при отгрузке товаров клиенту, схема отгрузки товаров для данного магазина однофазная. В этом случае в строке товара в поле Количество будет отображен значок Статусы серии. Значок красного цвета означает, что серии еще не введены. Для ввода серии нужно из поля Количество открыть форму Ввод количества и серий товаров и заполнить её. После корректного ввода серии и количества, значок серии меняет свой цвет на зеленый.

Для отобранных товаров по умолчанию заполняются их текущие розничные цены. При наличии соответствующих прав Пользователь может вручную изменить цены в строках или заполнить их по другому виду цен с помощью команды из меню Цены в командной панели табличной части.

После подбора товаров и заполнения цен на странице Товары Пользователь должен рассчитать автоматические скидки с помощью команды Рассчитать скидки из меню Скидки (наценки) или из командной панели на странице Скидки. Кроме того, Пользователь может назначить дополнительные управляемые или ручные скидки.

Товары, для которых рассчитались скидки, и суммы скидок можно просмотреть на странице Скидки табличной части. Для каждого выбранного товара на странице Товары можно детально просмотреть информацию о примененных скидках с помощью команды Открыть информацию о скидках (наценках) для текущей строки из меню Скидки (наценки).

При проведении документа система анализирует считанную дисконтную карту и, если эта карта участвует в бонусной программе, то производится начисление бонусных баллов, которые отображаются на странице Бонусы табличной части.

На закладке Дополнительно указываются следующие параметры:

Параметры налогообложения - Учитывать НДС и Цена включает НДС;

Реквизиты доверенности, если товар покупает сотрудник организации-контрагента;

Счет организации-продавца (для оформления документов);

Адрес доставки, если необходимо организовать доставку проданного товара.

Документы, который можно оформить на основании документа Реализация товаров

На основании документа Реализация товаров можно оформить следующие документы:

· Расходный ордер - для оформления отгрузки по ордерной схеме;

· Приходный кассовый ордер с видом операции Поступление оплаты от покупателя - для регистрации оплаты покупки;

· Эквайринговая операция - для регистрации оплаты покупки, если оплата производится с помощью платежной карты;

· Возврат товаров от покупателя - для оформления частичного или полного возврата товара из данной покупки.

Реквизиты создаваемых документов заполняются значениями реквизитов документа-основания.

Частичная /полная оплата товара при передаче покупателю

Передача товаров покупателю регистрируется с помощью документа Реализация товаров, созданного на основании Заказа покупателя. Если планируется передача товаров без какой-либо оплаты, то чек можно напечатать из документа Реализация товаров и в этом случае признаку способа расчета будет присвоено значение "Передача предмета расчета без его оплаты в момент его передачи с последующей оплатой в кредит". Для печати чека нужно на закладке Фискализация данных указать кассу ККМ, на которой будет пробит чек.

Если же при передаче товаров покупателю производится частичная оплата за товары, необходимо на основании документа Реализация товаров создать Приходный кассовый ордер или документ Эквайринговая операция и зарегистрировать сумму, полученную от покупателя.
