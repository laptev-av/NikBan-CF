Типовые конфигурации
Розница, редакция 2.2 и Отчетность предпринимателя, редакция 2.0

 

 

Совместное использование конфигураций "Розница ред. 2.2" и " Отчетность предпринимателя ред. 2.0"

 

Цели совместного использования конфигураций

 

Программу "1С:Розница 8" можно использовать совместно с программой "1С:Отчетность предпринимателя 8". Синхронизация предназначена для предпринимателей и небольших фирм, которые применяют упрощенную систему налогообложения (УСН) и/или единый налог на вмененный доход (ЕНВД). При этом в программе "1С:Розница 8" будет вестись учет оперативной деятельности, а программа "1С:Отчетность предпринимателя 8" будет использоваться для подготовки и предоставления отчетности контролирующим органам.
Одновременное использование двух конфигураций имеет ряд преимуществ:

  • Использовать возможности специализированного сервиса обслуживания розничных продаж (Рабочее место кассира).
  • Вести учет операций товаров непосредственно в магазинах, в местах осуществления операций, оперативно исправляя ошибки и недочеты.
  • Отслеживать сроки подготовки отчетности и уплаты налогов и взносов;
  • Подготавливать и сдавать отчетность в электронном или бумажном виде, сдавать отчетность через Интернет;
  • Рассчитывать налоги и взносы, контролировать уплату налогов и взносов

Схема совместного использования двух конфигураций применима только при ведении несложного учета. В иных случаях для подготовки регламентированной отчетности рекомендуется использовать программу "1С:Бухгалтерия 8".

 

Порядок настройки обмена данными между конфигурациями

 

Учетные данные, накопленные в программе «1С:Розница», должны периодически передаваться в программу «1С:Отчетность предпринимателя», где они будут отражены в налоговом учете упрощенной системы налогообложения (УСН) как записи Книги учета доходов и расходов.

Настройка обмена производится с помощью мастеров настройки обмена в конфигурации "1С:Розница".

При первоначальной настройке обмена происходит попытка автоматической синхронизации, сопоставление существующих данных в обеих конфигурациях и совершается перенос несопоставленных объектов в конфигурацию «Отчетность предпринимателя». При последующих обменах между базами будут передаваться только вновь созданные или измененные объекты.

Для настройки синхронизации, необходимо начать ввести учет в конфигурации «Розница». Необходимо ввести справочную информацию: организации, банковские счета, признаки ведения учета по ЕНВД, УСН. Далее, необходимо создать новое (пустое) приложение «Отчетность предпринимателя» и настроить синхронизацию по необходимыми фильтрами.

Важно! Применение фильтров выгрузки

Принцип действия фильтров выгрузки таков, что новые настройки в момент создания обмена применяются для всех данных. При изменении настроек фильтра после создания обмена данных, они будут применены только для тех данных, которые были изменены после момента применения новых настроек. Поэтому рекомендуется тщательно продумать настройки фильтров при создании обмена данными.

Создание обмена в конфигурации Розница

 

Использование обмена данными определяется в настройках параметров учета (раздел Настройка и администрирование – команда Синхронизация данных). В форме настройки параметров синхронизации данных установите флаг Использовать синхронизацию данных, задайте Префикс информационной базы.

Откройте список Синхронизация данных, нажмите кнопку Настроить синхронизацию данных, в выпадающем меню выберите пункт Отчетность предпринимателя, ред. 2.0… При этом откроется первая страница помощника создания синхронизации данных. Для перехода к следующей странице помощника используйте кнопку Далее, для возврата к предыдущей – Назад. Выберите Шаг 1. Создать новую синхронизацию данных. На следующей странице необходимо выбрать место расположения второй информационной базы:

  • Подключиться ко второй информационной базе напрямую;
  • Подключиться ко второй информационной базе через обычные каналы связи.

 

При подключении напрямую необходимо указать параметры подключения ко второй информационной базе: расположение (На данном компьютере или на компьютере в локальной сети, либо На сервере 1С:Предприятия) и параметры аутентификации (Пользователь, Пароль).

 

При подключении через обычные каналы связи на трех следующих страницах выполните настройку транспорта сообщений между информационными базами:

 

  • Обмен данными через локальный или сетевой каталог;
  • Обмен данными через FTP-ресурс;
  • Обмен данными по электронной почте.

 

На странице Настройка параметров синхронизации данных введите наименование обмена данными (произвольное). Название желательно подобрать с учетом того, что и в конфигурации Отчетность предпринимателя по умолчанию настройка обмена будет названа так же. Нажмите на кнопку Изменить правила выгрузки данных и заполните настройки синхронизации:

 

  • Установите Дату начала выгрузки документов
  • При необходимости установите флаг «Отбор по организациям» и выберите организации, по которым будут фильтроваться выгружаемые данные.

 

Если по некоторым складам ведется деятельность по ЕНВД, то необходимо установить флаг Не передавать документы по ЕНВД. В этом случае в миграции не будет участвовать документы доходов и расходов, которые относятся к деятельности по ЕНВД.

 

Сохраните заданные ограничения, нажав на кнопку OK. Перейдите Далее к проверке заданных настроек обмена данными. На последней странице помощника создания настройки синхронизации данных нажмите кнопку Сохранить для сохранения файла настроек обмена, установите флаг Выполнить выгрузку данных сейчас и нажмите кнопку Готово.

 

 

Продолжение настройки обмена в конфигурации Отчетность предпринимателя

 

Если на первом шаге было выбрано подключение через обычные каналы связи, то необходимо продолжить настройку в конфигурации Отчетность предпринимателя.

 

В настройках конфигурации должен быть указан режим Использовать синхронизацию данных. Использование обмена данными определяется в настройках синхронизации данных (раздел Настройка - команда Синхронизация данных). В разделе Настройка нажмите кнопку Настроить синхронизацию данных, в выпадающем меню выберите Розница, редакция 2.2...

 

В открывшейся форме помощника создания обмена данными выберите Шаг 2. Продолжить настройку синхронизации данных, укажите путь к ранее сохраненному файлу настроек обмена, сформированному при настройке обмена в конфигурации Розница. Нажмите Далее и на двух следующих страницах выполните, если требуется, настройку транспорта сообщений между информационными базами. На последней странице помощника создания настройки синхронизации данных нажмите кнопку Готово. Настройка синхронизации завершена.

 

Фильтрация документов по ЕНВД

 

Если на этапе настройки синхронизации был установлен флаг Фильтровать документы по ЕНВД, то в миграции не будут участвовать документы, которые относятся к деятельности по ЕНВД.

 

Документ

Деятельность по ЕНВД

Отчет о розничных продажах

Если на дату документа для организации ведется учет по ЕНВД одновременно по всем складам из табличной части Товары, то считаем что данный документ относится к деятельности по ЕНВД.

Во всех других случаях документ не относится к деятельности по ЕНВД.

Приходный кассовый ордер

Если на дату документа для организации ведется учет по ЕНВД одновременно по всем складам первичных документов из табличной части Расшифровка платежа, то считаем что данный документ относится к деятельности по ЕНВД.

Во всех других случаях считаем, что документ не относится к деятельности по ЕНВД.

Особый случай:

Если в табличной части Расшифровка платежа указан хотя бы один Расходный кассовый ордер, то считаем что данный документ не относится к деятельности по ЕНВД.

Расходный кассовый ордер

Если на дату документа для организации ведется учет по ЕНВД одновременно по всем складам первичных документов из табличной части Расшифровка платежа, то считаем что данный документ относится к деятельности по ЕНВД.

Во всех других случаях документ не относится к деятельности по ЕНВД.

Особый случай:

Если в табличной части Расшифровка платежа указан хотя бы один Приходный кассовый ордер, то считаем что данный документ не относится к деятельности по ЕНВД.

Оплата от покупателя

платежной картой

Регистрация безналичной оплаты

Если на дату документа для организации ведется учет по ЕНВД одновременно по всем складам первичных документов из табличной части Расшифровка платежа, то считаем что данный документ относится к деятельности по ЕНВД.

Во всех других случаях считаем, что документ не относится к деятельности по ЕНВД.
