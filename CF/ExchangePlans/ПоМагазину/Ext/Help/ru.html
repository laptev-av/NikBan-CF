Обмен «По магазину»

  Назначение

    • Реализация распределенной розничной сети магазинов.

  Общее описание

При работе конфигурации «Розница» в распределенной розничной сети возникает потребность в построении распределенной базы данных между магазинами.

Центральный узел может работать как один из магазинов, а остальные магазины, входящие в состав центрального узла, могут быть территориально распределены.

Данное решение позволит обеспечить:

  • Разделение информации в разрезе магазинов.
  • Ведение собственных розничных операций в каждом из магазинов.
  • Независимость магазинов при их территориальном разделении.

  Передача данных

Из центрального узла в магазины мигрирует следующая информация:

  • Справочники
  • Документы(с ограничением по магазину)
  • Информативные остатки других магазинов

 

Из узла магазина в центральный узел мигрирует следующая информация:

  • Справочники
  • Документы

 

  Настройка обмена «По магазину»

   Предварительные действия

 

Обмен между базами может производиться по определенному расписанию с использованием регламентных заданий. Для того чтобы такую возможность можно было бы использовать в конфигурации «Розница» в настройках параметров учета на закладке Обмен данными необходимо указать пользователя для регламентных заданий в файловом режиме и интервал опроса регламентных заданий в файловом режиме. Если используется клиент-серверный вариант, то пользователя для регламентных заданий и интервал опроса указывать не надо.

Для того, чтобы данные опроса не мешали в реальной работе лучше выделить отдельного пользователя для выполнения регламентных заданий обмена.

  Информативные остатки

Информативные остатки предназначены для информирования пользователя о товарных остатках в других магазинах сети.

Для включения механизма формирования информативных остатков по магазинам в настройках параметров учета необходимо включить флаг «Обновлять данные по информативным остаткам магазинов регламентным заданием» и задать расписание регламентного задания: данные об информативных остатках в магазинах будут обновляться по этому расписанию.

  Настройка главного узла

Перед началом настройки обмена данными необходимо ввести информацию о главном узле. Первоначально в списке узлов хранится информация об основном узле информационной базы главного узла. Название этого узла первоначально не определено.

Отредактируйте информацию о главном узле. Задайте код узла и его наименование.

  Настройка параметров и создание узла магазина

При добавлении нового узла(соответствующего определенному магазину) можно задать следующие параметры:

  • Код узла обмена

Служит идентификатором при обмене

  • Наименование узла обмена
  • Список магазинов

В список добавляются магазины, для которых создается узел.

  • Настройка выгрузки информативных остатков

В узел магазина можно выгружать информацию об остатках товаров, которые имеются в других магазинах, объединенных в центральный узел. При этом можно выгружать остатки по всем магазинам (флажок Выгружать остатки товаров всех магазинов) или задать список магазинов, по которым нужно выгружать остатки (флажок Индивидуальная настройка списка магазинов).

Примечание. После заполнения информации об узле распределенной базы данных необходимо закрыть программу и войти в нее заново, для того чтобы была зарегистрирована структура распределенной базы данных.

Для создания базы данных магазина воспользуйтесь кнопкой Создать начальный образ.

  Создание настройки обмена данными

 

Для созданных узлов необходимо создать настройки обмена данными. Добавить настройку обмена данными можно через меню Сервис-Обмен данными-Настройки обмена данными - Распределенные ИБ по магазину.

В новой настройке обмена данными необходимо указать созданный узел обмена, способ передачи данных, расписание обмена данными.

 

  Выполнение обмена данными

Для интерактивного запуска обмена данными используйте пункт меню Сервис – Обмен данными – Выполнить обмен данными.

В автоматическом режиме можно осуществлять контроль за результатами выполнения обменов.

Контроль обмена данными производится с помощью монитора обмена (Сервис – Распределенная ИБ по кассам – Монитор обмена данными).
