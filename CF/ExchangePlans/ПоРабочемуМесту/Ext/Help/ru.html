Обмен «По кассе»

Назначение

  • Обеспечение возможности беспрерывного функционирования базы «Розница» с приемлемой производительностью в магазинах с большим количеством касс
  • Обеспечение независимости работы касс от средств связи с сервером.

Общее описание

При работе конфигурации «Розница» в магазинах, где установлено несколько касс, и на каждой кассе работает отдельный сеанс «1С:Предприятия», подключенный к серверу, производительность программы на кассах может быть снижена (либо вообще приостановлена) из-за следующих факторов:

  • Интенсивная работа других пользователей с базой;
  • Проблемы работы локальной сети;
  • Проблемы в работе с сервером (например, загрузка сервера из-за выполнения обмена с управляющей системой).

Основной целью данного механизма является устранение данных недостатков с помощью использования в магазинах механизма распределенных информационных баз, в котором каждой кассе будет соответствовать отдельный узел РИБ.

Данное решение позволит обеспечить:

  •  На кассах большую производительность по сравнению с многопользовательским режимом работы;
  •  Независимость работы касс от работы локальной сети;
  •  Независимость работы касс от работы сервера.

Передача данных

Кассир на своем рабочем месте по мере выполнения обмена данными может получать следующую информацию по:

  •  Номенклатуре, единицам измерения номенклатуры, характеристикам номенклатуры(а также свойствам и категориям)
  •  Ценам номенклатуры магазинов
  •  Скидкам
  •  Дисконтным картам
  •  Продажам по дисконтным картам
  •  Продажам по платежным картам
  •  Остаткам товаров в других магазинах
  •  Остаткам товаров на складах текущего магазина
  •  Денежным средствам, относящимся к его кассе
  •  Чекам, относящимся к его кассе
  •  Отчетам о розничных продажах, относящимся к его кассе.

Данная информация позволяет кассиру полноценно работать в своей локальной базе.

Из локальной базы кассы может передаваться информация по:

  •  Продажам по дисконтным картам
  •  Продажам по платежным картам
  •  Денежным средствам
  •  Чекам
  •  Отчетам о розничных продажах

Настройка обмена «По кассе»

Предварительные действия

Обмен между базами может производиться по определенному расписанию с использованием регламентных заданий. Для того чтобы такую возможность можно было бы использовать в конфигурации «Розница» в настройках параметров учета на закладке Обмен данными необходимо указать пользователя для регламентных заданий в файловом режиме и интервал опроса регламентных заданий в файловом режиме. Если используется клиент-серверный вариант, то пользователя для регламентных заданий и интервал опроса указывать не надо.

Для того, чтобы данные опроса не мешали в реальной работе лучше выделить отдельного пользователя для выполнения регламентных заданий обмена.

Информативные остатки

Информативные остатки предназначены для информирования пользователя о товарных остатках на складах и магазинах.

  • Информативные остатки по складам хранят информацию о товарных остатках на складах текущего магазина.
  • Информативные остатки по магазинам хранят информацию о товарных остатках в других магазинах розничной сети.

Информативные остатки по магазинам и складам могут быть переданы в узел кассы.

Для включения механизма формирования информативных остатков по магазинам в настройках параметров учета необходимо включить флаг «Обновлять данные по информативным остаткам магазинов регламентным заданием» и задать расписание регламентного задания: данные об информативных остатках в магазинах будут обновляться по этому расписанию.

Для включения механизма формирования информативных остатков по складам текущего магазина в настройках параметров учета необходимо включить флаг «Использовать обмен информативными остатками по складам» и задать расписание регламентного задания: данные об информативных остатках по складам будут обновляться по этому расписанию.
Настройка главного узла

Перед началом настройки обмена данными необходимо ввести информацию о главном узле, который будет являться инициатором работы с кассами. Главный узел будет обеспечивать передачу в кассы информации о продаваемых товарах и их ценах и получение информации о продажах товаров, оформленных в различных кассах, подсоединенных в качестве узла РИБ.

Первоначально в списке узлов хранится информация об основном узле информационной базы главного узла. Название этого узла первоначально не определено.

Отредактируйте информацию о главном узле. Задайте код узла и его наименование.

Создание настройки обмена

Дальнейшая настройка обмена данными производится с помощью помощника настройки обмена (Сервис – Распределенная ИБ по кассам – Помощник настройки обмена данными с узлом кассы).

Помощник позволяет последовательно и безошибочно задать все параметры обмена, создать узел кассы, настройку обмена данными для этого узла, задать расписание обмена, создать начальный образ информационной базы кассы(локальной базы).

Запустите помощник. На первой странице установите переключатель в положение «Создать новую настройку обмена». Нажмите «Далее».

На  странице "Параметры обмена данными" задайте кассу, для которой создается локальная база, начальную дату выгрузки документов(документы ранее этой даты выгружены не будут), и наименование узла обмена(любое удобное для Вас). Нажмите кнопку «Далее».

На  странице "Передача данных между информационными базами" установите переключатель в положение «Единый файловый ресурс», в поле ввода «Каталог» укажите каталог, который будет использоваться для обмена данными с локальной базой кассы.

Нажмите кнопку «Готово». На последней странице помощника можно задать расписание автоматического обмена, создать начальный образ локальной базы кассы(указав каталог на диске и нажав кнопку «Создать начальный образ»).

Настройка расписания обмена

При нажатии на гиперссылку «Создать новое расписание для настройки обмена» откроется форма настройки выполнения обмена.

Изменение настройки обмена.

Запустите помощник и установите переключатель в положение «Изменить существующую настройку обмена». Дальнейшая работа с настройкой не отличается от сценария, описанного ранее. В конце работы помощника настройка обмена будет сохранена.

Настройка параметров узла кассы

При настройке обмена через помощник при создании новой настройки обмена данными создается новый узел обмена «По кассе».

Просмотреть все узлы обмена по кассе можно через меню Сервис – Распределенная ИБ по кассам – Узлы РИБ по кассам.

Открыв узел плана обмена  можно изменить параметры, заданные в помощнике настройки обмена с узлом кассы.

Также для каждого узла можно задать режим выгрузки информативных остатков:

  • Выгружать информативные остатки по складам.

При установленном флаге в узел кассы выгружаются остатки товаров по складам текущего магазина. Отображаются в подборах (также в режиме РМК).

  • Выгружать информативные остатки по другим магазинам.

При установленном флаге в узел кассы выгружаются остатки товаров по другим магазинам. Отображаются в подборах (также в режиме РМК).

Для формирования остатков необходимо также задать использование этих механизмов в настройках параметров учета.

Важно!

В кассу ККМ не выгружается информация о текущих остатках товаров на складах магазина (информативные остатки представляют лишь информацию и не имеют отношения к контролю остатков). Поэтому перед началом работы в настройках рабочего места кассира нужно отключить контроль остатков товаров при закрытии чека.

В том случае если обмен кассы с сервером не производился продолжительное время, на кассе может скопиться большое количество чеков к выгрузке. После восстановления связи с сервером выгрузка этих чеков может занять продолжительное время и, следовательно, затормозить работу кассы.

Для того чтобы не допустить подобной ситуации, необходимо придерживаться следующей методики работы.

Сразу после восстановления связи между сервером и кассами рекомендуется на сервере отключить выполнение регламентных заданий по обмену с кассами.

Для этого необходимо открыть справочник Настройки выполнения обменов (Сервис – Обмен данными) и в диалоговом окне соответствующей записи о настройках выполнения обмена с кассами снять флажок Использовать расписание для выполнения обмена.

После этого на сервере рекомендуется поочередно инициировать вручную обмен со свободными от покупателей кассами через справочник Настройки обмена данными, используя кнопку Выполнить обмен.

После того как вручную будет произведен обмен со всеми кассами, нужно включить использование регламентных заданий по обмену с кассами.

Для интерактивной загрузки информации можно также использовать пункт меню Сервис – Обмен данными – Выполнить обмен данными.

В появившемся диалоговом окне необходимо выбрать узел обмена данными, соответствующий кассе ККМ, и нажать кнопку Выполнить обмен.

При выполнении обмена из кассы ККМ выгружается информация о пробитых на кассе чеках ККМ. После закрытия кассовой смены выгружается документ Отчет о розничных продажах.

Примечание. Для интерактивной загрузки информации из кассы ККМ в центральный узел также можно использовать пункт меню Обмен данными, если работа кассира производится с использованием рабочего места кассира (режим РМК).

Контроль обмена данными производится с помощью монитора обмена (Сервис – Распределенная ИБ по кассам – Монитор обмена данными).
