Настройки номенклатуры

В раздел включены настройки, управляющие правилами ведения номенклатуры. В разделе устанавливаются единицы измерения веса и объема, используемые по умолчанию при подстановке номенклатуры в табличные части документов, а также дополнительные колонки при отображении номенклатуры в документах. Панель открывается по ссылке Администрирование – Настройки номенклатуры.

Общие параметеры

· Единица измерения веса - Единица измерения веса, используемая по умолчанию.

· Единица измерения объема - Единица измерения объема, используемая по умолчанию.

· Характеристики номенклатуры - Возможность учета части свойств номенклатуры в отдельном справочнике. Например, обувь имеет общую линейку размеров. Размеры можно учитывать с помощью характеристик.

· Упаковки номенклатуры - Возможность указания количества одного и того же товара в разных единицах измерения (в коробках, паллетах и т.д.).

· Серии товаров - Учет номеров экземпляров товаров, номеров производственных партий, сроков годности и т.д.

· Разрешить дробное количество - возможность использования дробного количества для не весовой номенклатуры в настройках скидок и в документах

· Колонка при отображении номенклатур... Код…Артикул - Дополнительная колонка при отображении номенклатуры в формах документов.

· Полнотекстовый поиск при подборе товаров - Использование механизма полнотекстового поиска при подборе товаров.

Учет алкоголя

· Алкогольная продукция - Флажок Алкогольная продукция позволяет вести учет сведений, необходимых для заполнения деклараций об объемах розничной продажи алкоголя и спиртосодержащей продукции. При учете продаж согласно ЕГАИС необходимо также настроить обмен, указать дату начала работы с ЕГАИС и дату начала продаж. В некоторых случаях выгрузка немаркируемой алкогольной продукции в ЕГАИС может не производиться.

· Дата начала работы с ЕГАИС определяет дату начала взаимодействия с ЕГАИС и формирования Журнала регистрации розничных продаж алкогольной продукции по форме установленной приказом ФСРАР №164. По умолчанию это 01.01.2016. Для тестирования функционала и просмотра журнала по новой форме нужно изменить дату на любую прошедшую.

· Дата регистрации розничных продаж в ЕГАИС определяет дату, начиная с которой при оформлении розничной продажи система будет запрашивать код марки.

Для создания настроек обмена с ЕГАИС в списке необходимо нажать кнопку Создать и ввести параметры настройки: Выбрать Организацию, ИНН, КПП и магазин, в котором установлен данный экземпляр УТМ.

Код в ФС РАР - это идентификатор торговой точки в информационной базе ЕГАИС, присваивается в личном кабинете лицензиата при регистрации каждой торговой точки.

Адрес УТМ и Порт УТМ - это сетевые настройки компьютера, на котором развернут УТМ. Пользователь может указать указать, будет ли данная настройка использоваться для всех рабочих мест магазина или только на текущем рабочем месте. Настройка Загружать входящие документы определяет, будет ли пользователь на текущем рабочем месте работать с приходными ТТН ЕГАИС.

У пользователя появилась возможность выбрать Формат обмена, в котором будут формироваться исходящие в ЕГАИС пакеты данных. Для входящих пакетов поддерживаются оба формата данных.

При использовании клиент-серверной базы появляется дополнительная настройка, определяющая должен ли обмен с ЕГАИС исполняться в серверном или клиентском контексте. При работе пользователя в режиме веб-клиента выполнение обмена с ЕГАИС возможно только в серверном контексте. Для настройки регламентных заданий обработки запросов (см. далее) также необходимо исполнение в серверном контексте.

Обработка запросов ЕГАИС

При необходимости на закладке Исходящие запросы в ЕГАИС (Администрирование - Настройки номенклатуры - Учет алкоголя - Настройки обмена с ЕГАИС) можно просмотреть сформированные запросы и их статус и загрузить ответы на запросы с помощью команды Обработать ответы.

Для того, чтобы результаты запросов в УТМ поступали в конфигурацию автоматически, необходимо настроить регламентное задание Обработка ответов ЕГАИС в списке Регламентные и фоновые задания (Администрирование - Обслуживание - Регламентные задания).

Рекомендуется настроить ежедневное расписание с интервалом не менее 5 минут. Установить флаг Включено.

Штрихкодирование

· Использовать неуникальные штрихкоды товаров - флажок по умолчанию снят, т.к. проверка уникальности при введении штрихкода повышает оперативность и достоверность при оформлении всех операций с товаром. Если в программе установлено разрешение Использовать неуникальные штрихкоды, то в случае, когда результат поиска неоднозначен, список будет выведены в отдельную форму. Например, список товаров и подарочных сертификатов. Пользователь сможет выбрать нужный элемент по наименованию.

Если при заполнении товарной таблицы по штрихкоду будет найдена только дисконтная или служебная карта, то пользователю будет выведено соответствующее сообщение об ошибке.

· Префикс внутреннего штрихкода штучного товара (Н)

· Префикс штрихкода узла РИБ (00) - Шаблон формирования внутреннего штрихкода штучного товара 2H00NNNNNNNNZ; где Н-префикс внутреннего штрихкода штучного товара. 00-Префикс штрихкода узла РИБ. N-символ внутреннего кода в информационной базе. Z-контрольная сумма.

Штрихкодирование товаров на весах

· Префикс штрихкода весового товара на весах (2Х) - Шаблон формирования внутреннего штрихкода на весах 2XSSSSSWWWWWZ; где Х-префикс штрихкода. S- символ кода SKU. W-символ веса. Z-контрольная сумма.

· Префикс штрихкода штучного фасованного товара на весах (2Y) - Шаблон формирования внутреннего штрихкода на весах 2YSSSSSKKKKKZ; где Y-префикс штрихкода. S- символ кода SKU. К-символ количества. Z-контрольная сумма.

Коды SKU

· Коды товаров SKU назначаются в главном узле РИБ - SKU - идентификатор товарной позиции, в разрезе номенклатуры, характеристики и упаковки. Используемый при генерации штрихкодов на весах и идентификации товаров в кассах ККМ.

· Диапазон генерации SKU для весового товара - Нижняя и верхняя границы диапазона при генерации SKU.

· Автоматически генерировать SKU - Автоматически генерировать коды SKU при выгрузке товарных позиций на подключаемое оборудование.

Комиссионная торговля - Настройка позволяет использовать комиссионную схему торговли. После установления флага "Комиссионная торговля", становится доступной настройка "Приоритеты реализации товара", с предустановленным значением "Приоритет собстветнного товара". А также в разделе "Закупки" становится доступна подситстема "Комиссионная торговля". Приоритеты реализации товаров, выпадающий список позволяет определить в какой последовательности будут продаваться товары
