Анализ товарного остатка и формирование ордеров

Сервис Анализ товарного остатка и формирование ордеров предназначен для определения количества товара, который нужно передать в торговый зал со складов магазина. Сервис используется для формирования документов Ордер на перемещение товаров. Сервис запускается по ссылке Склад – Сервис - Анализ товарного остатка и формирование ордеров.

При запуске сервиса система анализирует средние продажи за указанный пользователем период, сравнивает их с остатками товара в торговом зале и рассчитывает количество к перемещению со складов с учетом указанного менеджером желаемого периода торговли и исходящего остатка.

Пример. В пятницу утром кладовщик продовольственного магазина собирается переместить товар со склада в торговый зал. Он вводит период анализа продаж – пятница и суббота прошлой недели, период торговли – 1 день, исходящий остаток – 0,5 дня. Обработка рассчитает потребность = Средние продажи за период * Количество дней торговли - (Текущий остаток в торговом зале - Исходящий остаток). Система выведет в форму товары, которые нужно подать в зал и которые имеются в наличии на складе.

Настройки сервиса

Пользователь заполняет параметры сервиса:

· Магазин – магазин, внутри которого будет перемещаться товар;

· Дата формирования ордеров – текущая дата;

· Получатель – торговый зал (отдел торгового зала) магазина, для которого будет рассчитана потребность в товаре;

· Отправители – склады, с которых будет осуществляться перемещение. Список отправителей заполняется после выбора магазина и склада-получателя всеми остальными складами выбранного магазина. По умолчанию отмечены все возможные отправители. Пользователь может снять флажки с некоторых складов. Тогда при расчете количества к перемещению эти склады учитываться не будут;

· Период анализа – период, за который система агрегирует и рассчитывает средние продажи;

· Продажи (в днях) – потребность в товаре будет рассчитана так, чтобы обеспечить продажи в течение указанного количества дней;

· Планируемый остаток (в днях) – потребность в товаре будет рассчитана так, чтобы планируемый исходящий товарный остаток обеспечивал продажи за указанное количество дней по истечении планируемого периода (страховой запас);

· Текущий остаток (в днях) – в результат выполнения сервиса будут выведены только те товары, по которым текущего остатка хватит не более, чем на указанное количество дней торговли.

Пользователь может также настроить дополнительно отборы и группировки товаров с помощью команды Перейти к отбору. По нажатии кнопки Рассчитать формируется список товаров.

Результат выполнения сервиса

Система рассчитывает рекомендуемое количество товара К перемещению, исходя из указанного пользователем количества дней торговли и количества дней продаж планируемого исходящего остатка. Номенклатура группируется по складу-отправителю. По умолчанию товарные строки сортируются по возрастанию показателя Остаток в днях торговли.

В сформированном списке кладовщик отмечает строки (по умолчанию отмечены все), по которым нужно сформировать документ перемещения. Документ формируется при нажатии кнопки Сформировать ордера в левом верхнем углу формы. В результате будет сформирован документ Ордер на перемещение по выбранным строкам. Пользователь может записать и провести документ. Для формирования следующего документа нужно пересчитать потребность.

Важно! В колонке К перемещению отображается потребность в товаре. Если он находится на нескольких указанных менеджером складах, то будет выведено несколько строк с остатками в разрезе складов, но с одинаковой потребностью. Рекомендуется формировать документы Ордер на перемещение последовательно с каждого склада, пересчитывая потребность после каждого документа.
