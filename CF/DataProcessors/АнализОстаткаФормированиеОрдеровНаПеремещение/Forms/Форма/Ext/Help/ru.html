Анализ товарного остатка и формирование ордеров

Сервис Анализ товарного остатка и формирование ордеров предназначен для определения количества товара, который нужно передать в торговый зал со складов магазина. Сервис используется для формирования документов Ордер на перемещение товаров.

При запуске сервиса система анализирует средние продажи за указанный пользователем период, сравнивает их с остатками товара в торговом зале и рассчитывает количество к перемещению со складов с учетом указанного пользователем желаемого периода торговли и исходящего остатка.

Пример.В пятницу утром кладовщик продовольственного магазина собирается переместить товар со склада в торговый зал. Он вводит период анализа продаж – пятница и суббота прошлой недели, период торговли – 1 день, исходящий остаток – 0,5 дня. Обработка рассчитает потребность = Средние продажи за период * (Количество дней торговли + Планируемый исходящий остаток в днях) - Текущий остаток в торговом зале. Система выведет в форму товары, которые нужно подать в зал и которые имеются в наличии на складе.

Сервис Анализ товарного остатка и формирование ордеров запускается по одноименной ссылке из подраздела Сервис в панели действий раздела Склад.

Настройки сервиса

Пользователь заполняет следующие параметры сервиса:

  • Магазин - магазин, внутри которого будет перемещаться товар;
  • Дата формирования ордеров - текущая дата;
  • Получатель - торговый зал (отдел торгового зала) магазина, для которого будет рассчитана потребность в товаре;
  • Отправители - склады, с которых будет осуществляться перемещение. Список отправителей заполняется после выбора магазина и склада-получателя всеми остальными складами выбранного магазина. По умолчанию отмечены все возможные отправители. Пользователь может снять флажки с некоторых складов. Тогда при расчете количества к перемещению эти склады учитываться не будут;
  • Период анализа - период, за который система агрегирует и рассчитывает средние продажи;
  • Продажи (в днях) - потребность в товаре будет рассчитана так, чтобы обеспечить продажи в течение указанного количества дней;
  • Планируемый остаток (в днях) - потребность в товаре будет рассчитана так, чтобы планируемый исходящий остаток после продаж закрывал указанное количество дней (страховой запас);
  • Текущий остаток (в днях) - в результат выполнения сервиса будут выведены только те товары, по которым текущего остатка хватит не более, чем на указанное количество дней торговли.

При запуске сервиса некоторые параметры могут быть заполнены по умолчанию согласно настройкам пользователя.

Пользователь может дополнительно настроить отбор товаров с помощью команды Перейти к отбору.

Результат выполнения сервиса

Система рассчитывает рекомендуемое количество товара К перемещению, исходя из средних продаж за период анализа и указанных пользователем количества дней торговли и количества дней, которые должен закрыть планируемый исходящий остаток. После расчета на экран выводятся только те товары, у которых текущий остаток меньше указанного пользователем (в днях торговли). Таким образом, сначала пользователь может отбирать товары с маленьким текущим остатком.

Если товара на складах-отправителях недостаточно для покрытия потребности (дефицит), то соответствующие строки будут выделены красным цветом.

Важно. Если товар к перемещению хранится на нескольких складах-отправителях, то такие строки будут отмечены флажком в колонке Важно. При этом одной товарной позиции будет соответствовать несколько строк с остатками в разрезе складов, но с одинаковой потребностью. Рекомендуется формировать документы Ордер на перемещение последовательно с каждого склада, пересчитывая потребность после каждого документа.

Ордер на перемещение формируется при нажатии кнопки Сформировать ордера в левом верхнем углу формы. Пользователь может отредактировать, записать и провести документ. Для формирования следующего документа нужно перерассчитать потребность.
